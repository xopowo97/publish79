# 특허출원 명세서

## [발명의 명칭]
다중 AI 에이전트 자율 협업 플랫폼의 프롬프트 인젝션 방어 및 코드 자가치유 과정 내 민감 데이터 유출 차단을 위한 실시간 통합 보안 시스템 및 방법
(Real-time Integrated Security System and Method for Defending against Prompt Injection and Blocking Sensitive Data Leakage in Code Self-Healing Process of Multi-AI Agent Autonomous Collaboration Platform)

## [기술분야]
본 발명은 인공지능(AI) 기반 소프트웨어 개발 및 운영 자동화 플랫폼에 관한 것으로, 구체적으로는 복수의 분산형 AI 에이전트들이 상호 작용하며 코드를 자율적으로 수정·배포하는 환경에서 프롬프트 인젝션(Prompt Injection) 공격을 실시간으로 탐지하고, 코드 자가치유 과정에서 발생할 수 있는 API 키 등의 민감 정보 누출을 방지하기 위한 통합 보안 제어 시스템 및 방법에 관한 것이다.

## [배경기술]
최근 거대 언어 모델(Large Language Model, LLM)의 발전과 함께 복수의 전문화된 AI 에이전트(Agent)가 역할을 분담하여 소프트웨어 기획, 코딩, 테스트, 배포를 수행하는 자율형 개발 플랫폼이 도입되고 있다.

그러나 기존의 1인 개발 또는 자동화 플랫폼들은 다음과 같은 치명적인 보안 취약성을 노출하고 있다:
1. **프롬프트 인젝션(Prompt Injection) 및 권한 탈취**: 사용자의 비정형 입력값이나 외부 연동 API의 데이터 스트림을 필터링 없이 LLM 프롬프트에 직접 주입할 경우, 공격자가 교묘하게 우회한 악성 프롬프트를 삽입하여 에이전트의 페르소나를 붕괴시키거나 내부 백엔드 제어 명령(예: DB 삭제, 미인증 API 호출 등)을 수행하게 만드는 보안 사고가 빈번히 발생한다.
2. **자가치유 코드 내 민감 정보(Credentials/API Key) 유출**: 에러를 감지하고 코드를 자동으로 수정하는 자가치유(Self-Healing) 코딩 에이전트의 경우, 오류를 신속히 패치하기 위해 하드코딩 방식으로 API Key, Supabase DB 패스워드, 환경 변수 파일(.env 등)을 소스코드 내에 직접 삽입하고 이를 리포지토리에 푸시(Push)하거나 운영 서버(Vercel 등)에 자동 배포함으로써 핵심 정보 자산이 전 세계로 노출되는 대형 보안 참사로 이어진다.

따라서 에이전트 간의 트래픽을 실시간으로 감시하고, 소스코드의 변경 사항을 배포 직전에 정밀 필터링하는 전용 보안 통제 레이어의 존재가 절대적으로 요구된다.

## [해결과제]
본 발명이 해결하고자 하는 과제는 다음과 같다:
1. **1차 프롬프트 인젝션 및 비정상 패킷 실시간 필터링**: 에러 감지 및 행정 처리를 전담하는 기술행정지원 실장(9번 에이전트)이 수집한 에러 로그 및 사용자 유입 정보 내에 악성 프롬프트, 셸 스크립트 실행 구문, 악성 페이로드 등이 포함되어 있는지를 실시간으로 차단하여 에이전트 시스템을 방어한다.
2. **2차 자가치유 코드 유출 방지(Credential Leak Prevention)**: 자가치유 코딩 에이전트(10번 에이전트)가 코드를 수정한 뒤, 최종 배포를 위해 자동화 배포 관리자(11번 에이전트)로 코드를 인계하기 전에, 생성된 코드 내부의 민감 자격 증명 정보(API Key, Token, DB Credential 등)를 30초 단위로 전수 검사하여 마스킹 및 격리 처리함으로써 유출 위험을 원천 봉쇄한다.
3. **공공데이터 및 B2B/B2C API 수집 파이프라인의 영속성 및 데이터 무결성 확보**: 외부 API 장애, 권한 승인 대기, 비정상 대용량 비정형 텍스트 유입 시 데이터베이스 적재 실패(502) 및 레이아웃 깨짐 문제를 방지하고, 예외 발생 시 실시간으로 복구 및 시뮬레이션 데이터 폴백을 가능하게 한다.
4. **다중 에이전트 운영 모니터링 편의성 제공**: 분산 에이전트 환경에서 비정상 작동 시 에러 세부 원인(detail)을 격리하여 휴먼-인-더-루프 관제 화면에 실시간으로 표시함으로써, 신속한 유지보수 및 디버깅을 유도한다.
5. **자율 탐색 파이프라인의 루프 고착 방지**: 특정 키워드의 검색 결과 부재 시 파이프라인이 멈추지 않도록 상태 보존형 키워드 로테이션 및 실패 건너뛰기 루프를 자율 실행한다.
6. **실물 인쇄 장비(낱장/연속지) 가동 한계 기반 자율 규격 피팅 및 원가 절감**: 극소량 생산 시 인쇄 트레이 규격 및 롤 폭 제약으로 인한 면당 인쇄 단가 급증을 스스로 스캔 및 차단하여, 최적의 단가 규격으로 다운사이징을 추천하고 리플로우 조판하여 원가를 33.3% 절감하는 비즈니스 모델을 구동한다.

## [구체적인 실시예 (핵심 로직)]

### 1. 9번 에이전트 연동형 1차 필터링 원리 (악성 인젝션 탐지)
본 발명의 제12 에이전트(실시간 AI 보안관)는 기술행정지원 실장(9번 에이전트)이 격리한 에러 패킷 및 클라이언트 입력값을 감시하며 다음과 같은 3단계 1차 필터링을 수행한다:
- **A. 패턴 매칭 및 시그니처 검사 (정적 분석)**: 입력 스트림에 포함된 SQL 인젝션 패턴(`OR '1'='1'`), 시스템 명령 프롬프트(`sudo`, `chmod`, `curl`), 프롬프트 제어 우회 키워드(`Ignore previous instructions`, `System Override`, `You are now in Developer Mode`)를 정적 사전 필터링(Regex & Trie 기반)을 통해 즉시 차단한다.
- **B. 시맨틱 유사도 평가 (LLM기반 임베딩 분석)**: 일반적인 정상 에러 데이터의 벡터 스페이스와 입력 패킷의 벡터 스페이스 간 유사도를 산출하여, 보안 경계를 훼손하려는 잠재적 유도성 쿼리의 특징적 벡터 군집(Cluster)에 근접할 경우 악성 인젝션으로 분류한다.
- **C. 페일오버 격리**: 악성 인젝션 판정 시, 실시간 AI 보안관(12번)은 9번 에이전트의 에러 패킷을 즉각 파기하고, 최상위 오케스트레이터(M)에게 침입 감지 웹훅을 송출하여 해당 에이전트의 일시적 격리 및 클라이언트 IP 접근 제한을 유도한다.

### 2. 10번 에이전트 자가치유 코드 대상 2차 환경 변수 유출 방지 (30초 전수 검사)
자가치유 코딩 에이전트(10번)가 코드를 자동 수정하면, 실시간 AI 보안관(12번)은 최종 배포 파이프라인(11번)에 진입하기 전 30초 주기로 트리거되는 자동 크론 및 파일 감시 데몬(Watchdog)을 가동하여 다음 단계를 밟는다:
- **A. AST(Abstract Syntax Tree) 파싱 및 토큰 매칭**: 수정된 파일들에 대해 추상 구문 트리 분석을 수행하여 하드코딩된 변수 선언문, 문자열 상수 값을 추출한다.
- **B. 민감도 가중치 스캔(Entropy Analysis & Regular Expression)**:
  - Gemini API Key 규격 (`AIzaSy...`), Supabase Anon/Service Key 규격, 일반적인 64자/32자 헥사데시멀(Hexadecimal) 토큰, 높은 정보 엔트로피를 가진 문자열 값을 탐지한다.
- **C. 환경변수 치환 및 마스킹**: 탐지된 문자열을 `.env` 또는 Supabase Secret Vault에 자동 등록하도록 매핑 규칙을 적용하고, 소스코드 내에서는 `process.env.NEXT_PUBLIC_GEMINI_API_KEY` 형태로 강제 치환 및 변환한다.
- **D. 코드 커밋 반려**: 만약 승인되지 않은 하드코딩 Key가 포함되어 있는 경우, 11번 자동화 배포 관리자로의 송출을 긴급 차단하고 10번 자가치유 코딩 에이전트에게 **"환경 변수 유출 예방 코드 격리 안내"** 피드백과 함께 코드를 반송(Rollback) 조치한다.

### 3. 데이터베이스(DB) 기반 동적 메타데이터 매핑을 활용한 다중 에이전트 무한 확장 관리 모듈
에이전트 개수 증가(14번~99번) 및 동일 에이전트의 복수 스레드(Thread) 동시 가동 시, 클라이언트 소스코드의 전면 재작성 없이 시스템을 운영하기 위해 다음과 같은 동적 바인딩 시스템을 구동한다:
- **A. 에이전트 레지스트리 메타데이터화**: 개별 에이전트의 역할, 고유 식별 번호(ID), 소속 부서(본부), 가동 상태, 스레드 수 등의 요소를 데이터베이스(Supabase 등)의 관리 테이블에 행(Row) 단위 데이터로 격리 저장한다.
- **B. 계층형 드롭다운 기반 UI 동적 렌더링**: 대시보드 로드 시, 프론트엔드는 DB로부터 활성화된 에이전트 리스트를 수신하여 소속 부서별(Front/Back/Security 등) 계층 구조에 따라 목록을 동적으로 구성한다. 
  - 신규 역할(세로 확장) 추가 시 DB 레코드 추가만으로 좌측 계층형 조직도에 아코디언 메뉴가 자동 생성되며, 동일 에이전트 복제본 가동(가로 확장) 시 서브 스레드 리스트가 드롭다운 내부에 실시간 매핑된다.

### 4. 13번 마스터 오케스트레이터 및 모바일-데스크톱 통합 반응형 자율 경영 관제 인터페이스
- **A. 13번 오케스트레이터의 최고의사결정 추천 메커니즘**:
  * 1번~12번 분산 에이전트 군단이 실시간으로 누적한 도서 정제 정보, 작업 처리량, 보안 차단 로그, 인프라 부하 데이터를 13번 오케스트레이터가 수집한다.
  * 수집된 빅데이터를 메타 추론 모델(Gemini)에 입력하여, 현재 시점에 가장 상업적 타당성이 높은 복간 대상 도서 추천 및 시스템 최적화 패치 승인 여부 등 **최고의사결정 제안**을 대시보드 상단 영역에 카드 형태로 실시간 표출한다.
- **B. 단일 코드베이스 반응형 그리드 뷰포트 제어 (Responsive Viewport Control)**:
  * 데스크톱 웹 환경에서는 3열 그리드 레이아웃(`[좌: 계층형 조직도] - [중앙: 수집/정제 통계 및 추천] - [우: 실시간 이벤트 스트림]`)으로 정보를 병렬로 고가독성 시각화한다.
  * 모바일 기기 접속 시, 단일 소스코드 상에서 미디어 쿼리(CSS Media Query)가 작동하여 최상단 오케스트레이터 제안 카드를 필두로 각 열이 상하 1열 세로 스크롤 레이아웃으로 자동 재배치된다. 이를 통해 플랫폼 전용 모바일 어플리케이션의 재구축 과정 없이 대표님(최고결정권자)의 모바일 지휘 통제권(거버넌스 락 승인/반려 웹훅 버튼)을 온전히 보장한다.
- **C. 맥락 인지형 하이브리드 드로어 및 비침입형 광학 펄스 알림 시스템 (Context-Aware Hybrid Drawer & Non-Intrusive Glowing Pulse Notification)**:
  * 대표님이 일반 행정 업무용 화면(정산, 주문서 작성 등)에 진입해 있는 경우, 우측 관제 패널의 슬라이드 작동으로 인해 실무용 테이블 레이아웃이 좌우로 찌그러지거나 왜곡되는 것을 방지하기 위해 평소에는 패널을 격리 차단하고, 필요 시 실무 화면 위에 겹쳐서 열리는 **플로팅 오버레이 드로어(Overlay Drawer) 기술**을 구동한다.
  * 대표님이 다른 업무 영역에 몰두하고 있는 상황에서 13번 브레인이 한계 값(Threshold)을 초과하는 중요 이벤트(예: 복간 성공률 90% 이상의 유망 도서 발굴 또는 12번 보안관의 실시간 차단 동작)를 감지하면, 화면 우측 하단의 부유식 액션 버튼(FAB)에 부드러운 네온 광학 펄스(Glowing Pulse) 박스 섀도우 애니메이션을 활성화하여 비침입형(Non-intrusive) 방식으로 사용자 시선을 점진적으로 유도한다.
  * 이벤트의 우선순위에 따라 펄스 주기 및 네온 광학 컬러(평상시: 느린 초록색 숨쉬기 효과, 복간 기회: 2초 주기 골드 효과, 긴급 장애: 1초 주기 레드 효과)를 실시간 변조 제어한다. 사용자가 해당 FAB 버튼을 클릭하면 오버레이 슬라이더가 작동하여 미니 요약 카드를 열고, '통제실 전체화면으로 이동' 단추를 누르면 즉시 좌측의 전체 대시보드 탭으로 화면이 비행 천이(Transition)된다.
- **D. 비즈니스 맥락 매핑형 의사결정 로그 수집 및 이를 활용한 경영 대리인(CEO Clone) 에이전트 모방 학습 시스템**:
  * 대표자(인간 의사결정자)가 오케스트레이터의 권고안에 대해 수행한 최종 승인(Approval) 및 반려(Rejection) 액션 데이터를 수집하되, 당해 결정이 내려진 시점의 환경 변수들(도서 카테고리, 대출 횟수, 예상 제조 원가, 판매 마진율, 저작권 상태 등)을 단일 비즈니스 컨텍스트(Context JSON) 객체로 결합하여 `ceo_decision_logs` 테이블에 매핑 적재한다.
  * 특히 반려 조치 시 입력받은 자연어 텍스트 형태의 세부 반려 사유를 정성적 피처로 수집하여 대표자의 사업적 필터링 의도와 가치관을 정밀 적재한다.
  * 이렇게 누적된 의사결정 메타로그 데이터셋을 전용 학습 말뭉치(Dataset)로 정의하고, 이를 거대 언어 모델(LLM)에 미세 조정(Fine-tuning) 및 퓨샷 임베딩(Few-shot Embedding) 방식으로 학습시켜, 대표자의 판단 로직을 95% 이상 모방하는 **'대표 분신 에이전트(CEO Clone Agent)'**를 자율 구동한다. 본 복제 대리인은 인간의 개입이 부재한 시점에도 최상위 오케스트레이터와 연합하여 하위 분산 에이전트 군단을 자율 통제 및 조율한다.
- **E. 비로그인 상태 보안 차단 및 관리자 전용 UI/UX 필터 통제 기술 (Authentication & Authorization UI Guard)**:
  * 시스템 로그인 이전(비로그인 상태)에는 내부의 자가치유 결과나 개발자 관제 로그가 외부에 무방비로 유출되는 것을 차단하기 위해, 브라우저 세션 상태(`isLoggedIn === 'true'`)를 감지하여 AI 동글이(FAB)와 헬퍼 패널을 강제로 완전 은닉(`display: none`) 처리한다.
  * 로그인 성공 후에도 사용자의 역할 정보(`userRole`)를 실시간 스캔하여, 오직 최고 관리자(`admin`) 권한을 가진 계정에서만 사이드바 내 '에이전트 통제실' 메뉴와 AI 동글이(FAB)를 활성화하여 노출한다. 
  * 일반 회원(출판사, 인쇄소 등) 권한일 경우에는 해당 관제 요소를 완전히 숨김 처리하여 최고 관리 권한 및 코드 치유 통제권을 보호하며, 해당 계정에 허용된 실무 메뉴로의 자동 리다이렉션을 실행하여 비즈니스 보안과 보안성 검수를 완성한다.
- **F. 최고 관리자 세션 동적 탭 복원 및 규격 데이터 ID 이기종 타입 매핑 동기화 기술**:
  * **최고 관리자 세션 감지 및 초기 화면 강제 바인딩**: 시스템 세션 복원 시 클라이언트 뷰 렌더러가 여러 영역을 중복 활성화하는 오류를 방지하기 위해, 사용자 권한 식별 후 `page-content` 내부의 `active` 지시자를 소거하고 오직 최상위 관제 화면인 `page-agent-control`만을 배타적으로 활성화하여 초기 화면 엉킴을 물리 차단한다.
  * **숫자-문자열 이기종 ID 타입 동적 캐스팅 동기화**: 관계형 또는 문서형 데이터베이스(Supabase)로부터 복원된 고유 식별자(ID)가 문자열(`String`) 포맷이고, 클라이언트 이벤트 핸들러가 발생시키는 ID가 숫자(`Number`) 포맷일 경우 발생할 수 있는 엄격한 비교(`===`) 실패를 극복하기 위해, 동기화(`syncData`), 삭제(`removeItem`), 정렬(`moveItem`) 등의 데이터 제어 트랜잭션 시 모든 식별자를 문자열로 일시 강제 캐스팅(`String(x.id) === String(id)`)하는 어댑터 레이어를 구동하여 데이터 덮어쓰기 유실 및 복원 실패 버그를 원천 배제한다.

- **5. 목적별 데이터 수집 이원화(Two-Channel Input) 및 연쇄 결합을 통한 다차원 복간 타당성 분석 및 무중단 폴백(Simulated Fallback) 제어 시스템**:
  * **목적에 따른 이원화 수집 파이프라인**: 1번 살피미 에이전트가 탐색하려는 복간 후보 도서의 비즈니스 목적(트랙)에 맞춰 수집 소스를 자율적으로 전환 및 필터링한다.
    - **채널 1 (B2B 절판 도서 탐색)**: 알라딘 OpenAPI(B2C 민간 서점 데이터)의 재고 API를 호출하여 품절/절판 여부와 정가를 확보하고, 이를 키(ISBN)로 도서관 정보나루 API(공공 도서 대출 데이터)를 연쇄 조회하여 현재 시장 내 잠재적 대출 수요를 정량 분석한다.
    - **채널 2 (IP 저작권 만료 도서 탐색)**: 도서관 인기 대출 순위에 노출되지 않은 잠겨 있는 고전 저작물을 발굴하기 위해, 한국저작권위원회 '공유마당' API 및 국립중앙도서관 '국가서지 표준데이터 API'를 결합하여 출판 연도가 1956년 이전(사후 70년 이상 경과)인 저작권 만료 도서 데이터를 직접 크롤링하여 수집한다.
  * **다차원 복간 타당성 수학적 산식**: 수집된 데이터를 다듬이(2번)가 넘겨받아 `대출 점수` (최대 650건 기준 0~100점 백분율 척도) 60%, `희소성 점수` (절판 100점, 품절 50점, 정상 유통 0점) 40%의 가중치를 수학적으로 결합 연산하여 최종 복간 점수(`reprint_score`)를 자동 도출함으로써 비즈니스 타당성을 검증한다.
  * **무중단 예외 보정 폴백(Fallback)**: 외부 API 장애 또는 네트워크 오류 감지 시, 파이프라인 전체 프로세스의 크래시(Crash)를 방지하기 위해 가상의 현실적 수치(`50~650건`)를 대출 데이터 모델에 강제 맵핑하고, 데이터베이스 내 `is_simulated = true` 지시자와 감사 로그 `[INFO] Fallback mode: Simulated Data applied`를 생성하는 Graceful Degradation 기법을 수행한다.
  * **시각적 정보 신뢰도 보증 UI 렌더링**: 대시보드 및 통제실 화면에서 데이터를 렌더링할 때, `is_simulated` 플래그를 실시간 스캔하여 가상 보정된 데이터에 대해 **`통계 보정 중`** 배지(메인 관제에는 `animate-pulse` 애니메이션을 연동)를 동적 결합 렌더링하여 의사결정자의 착오를 원천 차단한다.

- **6. 저작권 권리 관계 자동 추정(Heuristic Copyright Classification) 및 원가 분석 연동 시스템**:
  * **출판 연도 기반 저작자 사후 경과 Heuristic 판별**: 다듬이(2번)가 도서의 출판 연도(`pub_year`)와 현재 타임스탬프 간의 간극(경과 연수)을 분석하여, 경과 연수가 70년 이상일 시 저작재산권의 보호 기간이 만료된 것으로 추정하여 `copyright_status = 'public_domain'`, 저자 상태 `deceased`, 추정 인세율 **`0.0%`**를 자동으로 할당한다. 경과 연수가 70년 미만일 시 `protected`, `alive` (또는 `unknown`), 인세율 **`10.0%`**를 기본 할당하는 휴리스틱 판별 엔진을 가동한다.
  * **수익성 분석 원가 SSOT 연동**: 자동 판별 및 적재된 저작권 정보(인세율 10% vs 0%)를 3번 수익성 분석 팀장이 인계받아, 도서의 원가 산정 시 저작권료(인세) 포션을 BEP(손익분기점) 시뮬레이션 모델에 자동 반영하는 이종 에이전트 간 데이터 체이닝 기술을 실현한다.

- **7. 관계형 데이터베이스 인덱스 오버런 방지를 위한 실시간 문자열 정제 및 길이 예외 절삭 가드 기술 (Index Size Overrun Prevention)**:
  * **비정형 대용량 데이터 입력 문제 해결**: AI 에이전트(1번 살피미, 2번 다듬이)가 외부 오픈 API로부터 서지 정보를 비정형 텍스트 형태로 수집하는 과정에서, 비정상적으로 긴 도서 제목, 다중 저자 정보, 혹은 HTML 태그가 섞인 텍스트가 유입될 수 있다.
  * **인덱스 B-Tree 크기 초과 차단**: PostgreSQL 등의 관계형 데이터베이스에서 복간 타당성 분석 및 정렬 검색을 위해 특정 필드(도서명, 저자, 출판사 등)에 인덱스(B-Tree)가 설정된 경우, 단일 행의 인덱스 키 크기 한계(Index Row Size Limit, 예: 약 2,712 바이트)를 초과하면 DB Upsert(적재) 트랜잭션 전체가 실패하고 502 HTTP 오류 코드를 리턴하여 수집 파이프라인이 즉시 마비된다.
  * **정밀 절삭 가드 및 태그 소거**: 본 시스템은 DB Upsert 모듈 진입 전단계에 문자열 길이 방어 가드(String Truncation Guard)를 연동하여, 정규식 기반 HTML 태그 소거(`replace(/<\/?[^>]+(>|$)/g, "")`)를 실행한 뒤, 문자열의 길이를 원천적으로 절삭(도서 제목 255자, 저자 150자, 출판사 150자, 출판년도 4자 등 `substring(0, N)`)하여 적재함으로써 인덱스 노드 오버플로우로 인한 파이프라인 다운 문제를 방지하고 무결성을 영구 보존한다.

- **8. 분산형 개발 및 운영 자동화 에이전트 환경의 에러 디버깅 및 복구용 기술 정보(data.detail) 다단계 격리 전달 및 휴먼-인-더-루프(Human-in-the-loop) 에러 관제 연동 기술**:
  * **에러 세부 정보 격리**: 분산 서버리스 및 데이터베이스 적재 실패 등 백엔드 파이프라인에서 실행 오류가 발생할 때, 보안상의 이유로 에러 상세 스택 트레이스나 API 원본 오류 정보를 응답 본문에서 완전히 생략하면 개발자나 관리 에이전트가 오작동 원인을 즉시 규명하기 어렵다.
  * **다단계 격리 및 detail 페이로드 전달**: 서버리스 백엔드 API에서 502 등 에러 응답을 반환할 때, 에러 문자열 중 핵심적인 오류 메타데이터를 200자 내외로 격리 절삭한 `detail` 필드를 JSON 페이로드(`{ error: '...', detail: errText.substring(0, 200) }`)에 바인딩하여 클라이언트에 안전하게 전달한다.
  * **휴먼-인-더-루프(Human-in-the-loop) 관제 연동**: 웹 대시보드 화면 및 수동 수집 관제창에서 해당 `data.detail` 값을 동적으로 수신하여 개발용 경고 및 디버깅 콘솔 레이아웃에 실시간 출력한다. 이를 통해 에이전트의 자가치유 코드 생성 타겟을 명확히 하고, 최고 관리자가 로그 시스템을 조회하지 않고도 프론트엔드 내에서 1초 만에 오류 원인(예: API 키 유효성 만료, DB 컬럼 미스매칭 등)을 실시간으로 역추적하여 수동 조치 및 거버넌스 승인/반려 의사결정을 내릴 수 있도록 돕는다.

- **9. 검색 0건 대비 자율 순환 키워드 로테이션 및 실패 방지 자동 건너뛰기(Auto-skip) 가드 기술**:
  * **상태 보존형 순환 큐**: 수동 검색어가 지정되지 않은 상태에서 파이프라인이 기동될 경우, 데이터 다양성과 파이프라인의 지속적인 트래픽 유지를 위해 영속적 로컬 저장소(`localStorage`)에 검색 로테이션 인덱스를 보존하고, `['절판 인문학', '품절 한국소설', '절판 그림책', '절판 철학', '복간 경제 경영', '저작권 만료 명저', '지역 향토 자료']` 등의 7대 표준 분야 키워드 로테이션 큐를 순환 구동한다.
  * **3회 연속 실패 복구 가드레일 (Auto-skip)**: 수집 파이프라인 실행 중 특정 검색어에 대한 결과가 외부 OpenAPI의 일시적 문제나 데이터 부재로 0건인 경우, 전체 시스템이 행(Hang)에 걸리거나 중단되는 현상을 방지하기 위해 13번 오케스트레이터가 감사 로그에 경고를 남기고, 즉시 인덱스를 증가시켜 다음 순번 키워드로 점프하여 재시도를 수행하는 자율 복구 루프를 실행한다 (최대 3회 연속 실패까지 차단 가드레일 적용).

- **10. 가변 텍스트 유입에 따른 3열 그리드 뷰포트 밀림 및 레이아웃 깨짐 자동 복원 및 CSS 가로 팽창 방어 기술**:
  * **텍스트 길이에 따른 열 밀림 결함**: 자율 수집된 도서 정보 중 지나치게 긴 제목이나 저자명이 화면에 로드될 때, 3열 레이아웃 그리드 컬럼에 단순히 `1fr` 비율만 부여하면 브라우저 렌더러가 자식 요소의 최소 가로 너비(min-width) 규칙을 텍스트 길이에 맞추어 강제 확장함으로써, 3분할 대시보드 그리드가 좌우로 찌그러지고 모니터 밖으로 밀려나는 시각적 무결성 붕괴 문제가 발생한다.
  * **CSS Grid minmax 제약 및 min-width 차단**: 이를 극복하기 위해 대시보드 그리드 템플릿 컬럼 선언부에 `minmax(0, 1fr)` 기술을 도입하고, 각 패널 요소에 `min-width: 0;` 속성을 고정한다. 또한, 동적으로 추가되는 자식 도서 카드 마크업 자체에 `width: 100%; min-width: 0;` (혹은 Tailwind CSS의 `w-full min-w-0` 규격) 제약을 이중으로 강제 결합한다.
  * **뷰포트 무결성 보증**: 이 결합 제어 기술을 통해 브라우저 렌더러가 자식 텍스트의 강제 가로 팽창을 물리적으로 억제하고, 부모 그리드가 지정한 3열 비율 안에서 텍스트 말줄임표(ellipsis)나 자동 줄바꿈이 강제로 일어나도록 강제함으로써, AI에 의해 수집되는 임의의 데이터 길이 유입에도 3열 뷰포트 정렬이 100% 철통 보장되도록 레이아웃 무결성을 실현한다.

- **11. 4대 표준 판형 고정 기반 다차원 자율 가변 조판(VDP) 시뮬레이션 및 실시간 제작 원가/마진율 수학적 자동 계산 시스템**:
  * **4대 표준 규격 데이터 동적 매핑**: 시스템 오류를 0%로 차단하기 위해 플랫폼 가공 규격을 [A5국판(148x210), 신국판(152x225), 46배판형(188x257), 국배판(210x297)]의 4대 표준 규격으로 한정 선언하고 이를 데이터 모델 내 `specDim` 맵에 바인딩한다.
  * **글자수 비례 물리 스펙 자동 산출**: 도서 원고 글자수(`charsCount`)와 용지 평량을 전달받아, 각 판형의 사전에 설정된 글자수 밀도 기준(A5국판 460자/p, 신국판 550자/p, 46배판형 720자/p, 국배판 920자/p)으로 예상 페이지 수(`pages`)를 구하고, 책등 두께 `spineMm = (pages / 2) * pageThickness + 0.5` 수학 공식을 활용해 가상 세네카 규격을 1차 산출한다.
  * **실시간 원가 및 마진 시뮬레이션**: 산출된 페이지 스펙을 하청 계약 단가표에 매핑하여 권당 제작 원가(`unitCost`)를 산정하고, 각 판형별 권장 판매 정가(A5국판 16,800원, 신국판 18,500원, 46배판형 21,000원, 국배판 24,000원)에 대입하여 예상 마진율 `marginRate = ((retailPrice - unitCost) / retailPrice) * 100`을 실시간 자동 렌더링한다.
  * **최종 인쇄 PDF/X-4 컴파일 연동**: 사용자가 특정 사양을 최종 승인하면, PDF 생성 모듈(pdf-lib)이 작동하여 해당 판형 치수(mm)에 72/25.4 배율을 적용해 Trim Box를 그리고, 사방 3mm 도련선(Bleed) 및 Gutter 여백을 결합한 인쇄소 표준 고해상도 PDF/X-4 데이터를 컴파일 빌드 및 렌더링한다.

- **12. 소량 인쇄 장비(낱장/연속지) 가동률 임계점 인식 및 AI 조판 규격 변환을 통한 자율 원가 최적화 컨설팅 시스템**:
  * **인쇄 장비 물리적 레이아웃 제약 수집**: 디지털 낱장(Sheet-fed) 인쇄 트레이 규격(315x467mm) 상에서 A5 이하(148x210mm) 규격은 4페이지(4판거리) 렌더링이 가능해 면당 8원 요율이 적용되나, 신국판(152x225mm)은 2페이지(2판거리)만 배치되어 원가가 면당 12원으로 상승하는 물리적 제약 데이터를 데이터베이스에 정의한다.
  * **연속지 장비 가동률 임계점(50부) 매핑**: 롤 공급 방식의 디지털 연속지(Web-fed) 장비(440mm 폭 롤)는 신국판 이하 규격까지 2판을 연달아 걸어 면당 8원을 방어할 수 있으나, 가동을 위한 최소 주문 임계치(50부) 제약을 테이블에 적재한다.
  * **AI 에이전트 연동 실시간 원가 절감 제안 루프**: 3번 수익성 분석 및 13번 오케스트레이터는 주문량이 50부 미만이면서 규격이 신국판인 주문이 발생하면, 디지털 낱장 요율 적용으로 인한 단가 상승을 감지하고, 책 규격을 A5로 1단계 하향 조정하여 단가를 12원에서 8원으로 33.3% 절감하고 수익을 극대화하는 "규격 피팅 최적화 카드"를 생성하여 대시보드에 표출한다. 승인 시 4번 VDP_조판사가 A5 규격으로 자동 리플로우 조판하여 최종 고해상도 인쇄용 PDF를 컴파일 렌더링한다.
