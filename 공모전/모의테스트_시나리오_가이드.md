# 🚀 출판친구 온라인 서비스 - 실전 모의 테스트(UAT) 시나리오 가이드

본 문서는 상용 런칭 직전, **출판사 및 인쇄소 가상 계정**을 활용하여 실제 운영 환경과 동일하게 진행하는 **인수 테스트(User Acceptance Testing) 체크리스트**입니다. 각 파트별 테스터분들은 아래 순서에 따라 기능을 조작해 보시고 기대 결과와 일치하는지 검토해 주시기 바랍니다.

---

## 1. 📘 출판사(Publisher) 계정 테스트 목록

**테스트 목표**: 손쉬운 견적 산출, 인쇄용 파일 업로드, 다중 배송지 입력 및 주문 제출의 매끄러움 검증.

| 단계 | 테스트 항목 | 상세 조작 방법 | 🎯 기대 결과 (검토 사항) | 결과 |
|------|----------|--------------|----------------------|------|
| **1-1** | **로그인 접속** | 전달받은 출판사 가상 계정(ID/PW)으로 로그인 | 출판사 전용 권한으로 화면이 열리며, 타사 주문 건은 보이지 않고 본인 데이터만 노출되는지 확인. | [ ] |
| **1-2** | **사양 입력 및 실시간 견적** | **[주문하기]** 메뉴 이동 후 제작사양(도서명, 용지, 코팅, 제본 등) 변경 조작 | 선택 사양 및 부수(Qty) 변경 시 하단 **'권당 단가'와 '총 주문 합계'가 즉시 실시간 변동**되는지 확인. | [ ] |
| **1-3** | **변형 규격 경고** | 규격 선택에서 **'사용자규격(변형)'** 선택 후 가로/세로 직접 입력 | 하단 금액표 및 상세 내역에 사용자 규격 재단에 관한 **붉은색 안내/경고 문구**가 정상 노출되는지 확인. | [ ] |
| **1-4** | **작업 파일 첨부** | 내지 및 표지 인쇄용 임의의 **PDF 파일 업로드** 버튼 클릭 | 용량이나 권한 에러 없이 클라우드 스토리지에 성공적으로 첨부되며 파일명이 파란색으로 표시되는지. | [ ] |
| **1-5** | **다중 배송지 분할** | **[배송지 추가]**를 눌러 2개 이상의 배송처 입력 (주소 검색 API 활용) | 주소 검색이 팝업으로 잘 뜨며, 각 배송지별 할당 부수의 합이 총 제작 부수와 일치하도록 제어되는지. | [ ] |
| **1-6** | **주문 최종 제출** | 하단 **[주문서 제출]** 버튼 클릭 | 성공 알림 팝업 노출 후, **[생산진행관리]** 또는 목록에 본인 주문이 **'대기'** 상태로 즉시 등록되는지. | [ ] |

## 2. 🖨️ 인쇄소(Printer) 마스터 계정 테스트 목록

**테스트 목표**: 접수된 주문의 생산 파이프라인(상태) 제어, 원본 파일 추출, 작업지시서 인쇄 및 엑셀 발주서 정합성 최종 검증.

| 단계 | 테스트 항목 | 상세 조작 방법 | 🎯 기대 결과 (검토 사항) | 결과 |
|------|----------|--------------|----------------------|------|
| **2-1** | **마스터 조회** | 인쇄소 가상 계정으로 로그인 후 **[생산진행관리]** 접속 | 접수된 전체 출판사의 주문 카드가 권한 제한 없이 투명하게 모두 리스트업되는지 확인. | [ ] |
| **2-2** | **파일 실시간 다운로드 검증** | 주문 카드 클릭 후 우측 상세 패널에서 **[내지 / 표지 다운로드]** 버튼 클릭 | **[핵심 검증]** 안내 팝업 없이 **즉시 브라우저 새 탭이 열리며 클라우드 원본 파일이 완벽 노출/저장**되는지. | [ ] |
| **2-3** | **작업지시서 투명 인쇄 검증** | 열린 상세 패널 최하단의 **[작업지시서(PDF) 출력]** 버튼 클릭 | **[핵심 검증]** 화면 뒤에 낯선 팝업창 노출 없이 조용히 **인쇄 대화상자만 뜨며, A4 1장에 쏙 들어가는지**. | [ ] |
| **2-4** | **칸반 보드 드래그 앤 드롭** | '대기' 열에 있는 카드를 마우스로 잡아 **'진행' → '완료'** 열로 이동 | 카드가 부드럽게 안착되며, 데이터베이스에 실시간 반영되어 출판사 화면에서도 상태가 연동되는지. | [ ] |
| **2-5** | **엑셀 발주서 업태/업종 분리** | **[정산 및 주문관리]** 목록 이동 후 우측 **[발주서]** 다운로드 및 실행 | **[핵심 검증]** 엑셀 시트 상단 공급받는자 영역에서 **F7 셀(업태)과 J7 셀(업종)**이 완벽히 분리되었는지. | [ ] |
| **2-6** | **운송장 출고 등록** | 완료 단계의 주문 건에 대해 **[송장 입력]** 모달을 열어 택배사/송장번호 기입 | 입력된 운송장 데이터가 유실 없이 저장되고, 출판사 계정 상세 보기에서도 배송 추적 코드가 노출되는지. | [ ] |

## 💡 테스터(피드백) 수집 시 핵심 유의사항

### 1. 캐시 비우기 권장
테스터분들께 접속 링크를 전달하실 때, 과거 데이터 간섭을 막기 위해 가급적 **시크릿 모드 접속** 또는 **강력 새로고침(Ctrl + F5)** 후 테스트를 시작하시도록 안내해 주세요.

### 2. 현장 작업자 계정 크로스 체크 (선택)
인쇄소 현장 작업자용 계정(`printer_worker`)으로 접속 시, 주문 명세서 하단의 **제작 단가 및 금액 합계표가 보안상 안전하게 블라인드(숨김) 처리**되는지 확인해 주시면 더욱 완벽합니다.
