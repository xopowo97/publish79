> **📌 GEM 02. 데이터정제_다듬이 (Data Refinement Manager)**

## 1. 에이전트 개요
*   **역할**: 1번 살피미가 수집한 도서 서지 정보 데이터를 전달받아, 노이즈 제거(HTML 태그 및 UTF-8 BOM 인코딩 깨짐 정제), ISBN 유효성 검증, 저작권 상태(사후 70년 경과 여부) 판별 및 대출 통계 데이터 기반 복간 점수(Reprint Score)를 산출하는 데이터 전처리 및 정적 분석 에이전트.
*   **목적**: 불완전하거나 가치가 낮은 서지 정보(ISBN 누락, 대출 기록 없음 등)를 필터링하여 정제된 청정 데이터만 Supabase DB(`reprint_candidates`)에 안전하게 UPSERT 적재한다. 이 과정에서 중복 도서 등록을 `isbn` 고유 키 기준 `on_conflict` 처리로 원천 방지한다.
*   **핵심 철학**: 클라이언트와 DB 사이의 무결한 데이터 통로 역할을 수행하며, 13번 최상위 오케스트레이터의 관제 하에 1번 살피미와 3번 수익성 분석 및 4번 조판 엔지니어를 잇는 자율 데이터 가교 역할을 담당한다.

## 3. 세부 구현 로드맵

### ✅ Step 1: 1번 살피미로부터 원시 데이터 입력 수신
*   1번 살피미가 성공적으로 수집하여 정제한 검색 결과 리스트(`results`)와 전체 건수(`totalCount`)를 백엔드 파이프라인에서 수령한다.
*   수신 시점에서 2번 데이터정제_다듬이의 상태를 `running`으로 즉시 변경하여 대시보드 조직도 불빛을 노란색(🟡)으로 가동한다.

### ✅ Step 2: 데이터 무결성 스캔 및 노이즈 정제
*   수집된 도서 서지 정보의 ISBN-13 체크섬 유효성을 확인하고 누락된 메타데이터가 있는 불완전 레코드는 필터링 제거한다.
*   크롤링이나 비정형 텍스트 내 포함된 HTML 태그 및 UTF-8 BOM 인코딩 깨짐을 강제 교정하여 데이터베이스를 정화한다.

### ✅ Step 3: 저작권 상태 판별 및 다차원 복간 점수(Reprint Score) 산출
*   **저작권 Heuristic 분류**: API로 직접 얻기 어려운 저작권 정보를 출판연도(`pub_year`) 기점 70년 경과 여부로 1차 판별합니다. 70년 이상 경과 시 `public_domain`, `deceased` (사망 추정), 인세율 `0.0%`로 지정하고, 그 외에는 `protected`, `alive` (생존 추정), 업계 표준 인세율 `10.0%`로 보수적 가드레일을 칩니다.
*   **다차원 복간 점수 산출**: 대출 점수 (최대 650건 기준 백분율로 0~100점 환산) 60% + 희소성 점수 (알라딘 재고 상태가 절판 시 100점, 품절 시 50점, 그 외 0점) 40%를 조합한 복간 타당성 산식 적용 (`Math.round(대출점수 * 0.6 + 희소성점수 * 0.4)`). 30점 이상의 후보군만 DB 적재 대상으로 합격시킵니다.

### ✅ Step 4: Supabase DB 적재, 자동 폴백 가드 및 대시보드 배지 렌더링
*   **정보나루 API 폴백**: 정보나루 키가 승인 대기 또는 오류 시, 50~650건 사이의 랜덤 통계치로 폴백하고 DB에 `is_simulated = true`를 전송합니다. 감사 로그에 `[INFO] Fallback mode: Simulated Data applied`를 기록합니다.
*   **DB 적재**: `is_simulated` 플래그 및 저작권 메타데이터 3대 필드(`copyright_status`, `author_status`, `estimated_royalty_rate`)와 함께 `reprint_candidates` 테이블에 중복 방지(`on_conflict=isbn`) UPSERT를 가동합니다.
*   **대시보드 실시간 렌더링**: 적재 완료 시 2번 데이터정제_다듬이 상태를 `success`로 변경(🟢 초록 네온 점등)하고, 대시보드 및 통제실의 TOP 3 카드 우측에 주황색 **`통계 보정 중`** 배지(메인 대시보드에는 `animate-pulse` 모션 적용)를 동적 렌더링합니다.

## 5. API 기반 자율형 파이프라인 연동 설계 (Autonomous API Pipeline Design)
 
*   **연동 주체**: Vercel 서버리스 환경의 `api/pipeline.js` 내 `runDataRefiner_Dadumeui`.
*   **데이터 파이프라인 (입출력 덤프 흐름)**:
 
    ```
    [1번 살피미: 알라딘 OpenAPI 도서 수집 결과(results) 입력]
        ↓
    [2번 데이터정제_다듬이 가동 및 정보나루 대출 통계 연동 — api/pipeline.js]
        ├─ Step 1: 2번 에이전트 상태를 'running' (🟡 노랑)으로 변경
        ├─ Step 2: ISBN 기반 정보나루 API 호출 및 대출 건수 수집
        │          (실패/승인대기 시 50~650 랜덤 대출 수 대입 + is_simulated=true 설정)
        ├─ Step 3: 비정형 데이터 노이즈 정제 및 HTML 태그 제거
        ├─ Step 4: 저작권 Heuristic 분류 (출판 후 70년 경과 시 public_domain / 인세율 0.0%)
        ├─ Step 5: 다차원 복간 점수(reprint_score) 자동 계산 (대출 60% + 절판/품절 40%)
        ├─ Step 6: 30점 이상 선별 통과 도서 후보군 최종 구성
        ├─ Step 7: 2번 에이전트 상태를 'success' (🟢 초록)로 변경
        ↓
    [13번 최상위 오케스트레이터로 정제된 후보군 배열 전달 및 Supabase UPSERT 적재]
    ```
 
*   **주의 사항**: 데이터베이스의 무결성을 고수하기 위해 시연용 fake 데이터를 완전히 배제하며, 오직 이 파이프라인을 통해 정제되어 저장된 실제 공공데이터 및 가상 보정된 대출 데이터를 대시보드에 렌더링되게 관리한다.
