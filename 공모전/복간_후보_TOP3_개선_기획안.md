# 복간 후보 TOP 3 장르별 동적 필터링 및 예약 펀딩 관제 개선 기획안

본 기획안은 대표님의 제안과 플랫폼의 상업적 생존력 극대화(락인 장벽 구축)를 바탕으로, 대시보드의 정적인 단점을 해소하고 **'선제작비 확보형 예약 펀딩 및 자율 마케팅 관제'** 시스템을 대시보드에 구현하기 위한 아키텍처 기획서입니다.

---

## 1. 배경 및 문제 인식

* **현상 및 한계**: 기존의 TOP 3는 전체 도서 중 점수가 높은 3권만 정적 노출되어 대시보드가 단조로워 보였습니다. 또한, 단순 절판도서 발굴 및 원고 파일 격리만으로는 출판사가 플랫폼을 우회해 타사(부크크 등)에서 인쇄하는 '정보 먹튀' 문제를 해결할 수 없습니다.
* **해결 제안**: 
  1. **정보나루 ➔ 알라딘 역발상 수집**을 통해 실제 독자 대출 수요가 검증된 장기품절 도서를 선 수집합니다.
  2. 독자의 돈으로 인쇄비를 먼저 모으는 **'선제작비 확보형 예약 펀딩 모델'**과 **'AI 숏폼 홍보 팩 무상 제공'**을 유기적으로 엮어 우회할 수 없는 마케팅 락인(Lock-in)을 형성합니다.
  3. 이를 대표님 관제실(`control.html`), 독자 몰(`store.html`), 출판사 포털(`index.html`)에 시각화하여 비즈니스 가시성을 극대화합니다.

---

## 2. 대시보드 및 플랫폼 연동 3대 설계안

```
[ 전체 대시보드 3열 레이아웃 구성 ]
┌───────────────────────────┬───────────────────────────┬───────────────────────────┐
│     에이전트 조직도       │  [전체] [소설] [인문] [경영]  │   실시간 에이전트 로그    │ (대안 C 퀵 칩)
│                           ├───────────────────────────┤                           │
│  - 🟢 13번 작동           │   장기품절 복간 TOP 3     │  - [살피미] 수집 완료     │
│  - 🟢 1번 작동            │  (펀딩 진행률 게이지 표시) │  - [다듬이] 정제 완료     │ (대안 A 필터링)
│  - 🟢 2번 작동            ├───────────────────────────┤                           │
│                           │ [7일 트렌드 / 장르 히트맵]│  - 8번 마케팅 영상 생성   │ (대안 B 공간 절약 탭)
│                           │  * 탭 스위칭으로 차트 변환│  - 대표님 승인 로그       │
│                           ├───────────────────────────┤                           │
│                           │  실시간 수집 도서 피드    │                           │
│└───────────────────────────┴───────────────────────────┴───────────────────────────┘
```

### 1) 12대 표준 카테고리 드롭다운 및 자동 필터링 대시보드
* 기존의 텍스트 타이핑 입력 방식 대신, 백엔드 v2.0이 지원하는 12대 표준 카테고리를 직관적으로 선택할 수 있는 **드롭다운(Select) 버튼**을 탑재하였습니다. (2026-06-07 완공)
* **카테고리별 자동 필터링 대시보드 (내일 구현 예정)**: 드롭다운에서 카테고리를 바꿀 때마다 대시보드 화면의 TOP 3 목록과 실시간 피드 목록이 자동으로 해당 카테고리 데이터만 필터링하여 노출되도록 연동합니다. (드롭다운에 `전체보기` 옵션도 포함)

### 2) 동적 상태 배지 개편
기존의 `통계 보정 중` 배지를 제거하고, 7번(영업)/8번(마케팅) 에이전트 및 예약 결제 단계를 보여주는 실무형 상태 배지로 개편합니다.
* 🟡 **`[B2B 협의 대기]`**: 7번 영업 에이전트가 출판사로 선제작비 확보형 예약 판매 제안서를 보낸 상태.
* 🔵 **`[예약 펀딩 진행 중 (XX%)]`**: 스토어(`store.html`)에 예약 페이지가 오픈되어 빌링키 예약 결제가 들어오고 있는 상태.
* 🟢 **`[제작비 확보 완료]`**: 목표 펀딩 부수(BEP)를 돌파하여 토스/포트원 일괄 결제가 정상 승인되고 인쇄소 발주 대기 상태인 안전 단계.

### 3) 16번 지휘_판다 제안 카드 내용 고도화
* 파이프라인 수집 성공 시 우측 하단 AI 헬퍼 패널에 **"선제작비 확보형 예약 펀딩 제안"** 메시지를 송출합니다.
* 제안 카드 내에 **예약 판매 목표 부수, 총 제작비 목표액, 예상 수수료(15%)** 등의 수치 정보를 자동으로 계산해 노출하며, 대표님이 `[🚀 예약 판매 개설 승인]`을 클릭하면 즉각 스토어에 페이지가 오픈되도록 설계합니다.

## 3. 사용자 및 시스템 진입 경로 (물리적 격리 보안)

1. **독자 (B2C)**: `store.html`로 진입하여 펀딩 도서에 **토스페이먼츠/포트원 빌링키 카드 예약 등록**을 진행합니다.
2. **출판사 (B2B)**: `index.html`에서 자사 계정으로 로그인하여 **`[정산 및 주문관리]`** 메뉴를 통해 펀딩 마감 수량, 제작비 정산 및 인세 지급 현황을 확인합니다. (보안을 위해 에이전트 통제실 `control.html` 접근 차단)
3. **대표님 (Admin)**: `index.html`에서 관리자로 로그인한 뒤, `[🤖 에이전트 통제실]` 메뉴를 클릭하여 `control.html`에서 전체 AI 에이전트 흐름을 모니터링합니다.

## 4. 향후 구현을 위한 액션 플랜

* **Step 1 (API 및 DB 연동)**: 
  * `/api/reprint-candidates` 서버리스 함수에 `category` 및 `funding_status` 데이터를 포함하도록 쿼리 보완.
  * `orders` 테이블 내에 `FUNDING` 상태 및 `billing_key` 연동 스키마 검토.
* **Step 2 (프론트엔드 UI 수정)**: 
  * `control.js` 및 `script.js`에 카테고리 퀵 칩 클릭 리스너 바인딩 및 펀딩 게이지 바(Gauge Bar) 렌더링 로직 추가.
  * 16번 지휘_판다 제안 카드 문구 고도화 및 3단계 상태 배지 클래스 바인딩.
* **Step 3 (스토어 및 PG 테스트 결제 연동)**:
  * `store.html`에 펀딩 진행률 UI 및 토스페이먼츠/포트원 샌드박스 테스트 빌링키 결제창 스크립트 결합.
  * 독자가 테스트 결제 성공 시 웹훅(Webhook)을 통해 DB 상태가 자동 업데이트되는 모의 결제 흐름 완성.
* **Step 4 (출판사 정산관리 연동)**:
  * 펀딩 성공 시 `orders` 테이블 수량 합산이 출판사 전용 포털(`index.html`)의 `정산 및 주문관리` 테이블에 정확한 원가/수익금으로 매핑 노출되는지 검증.
* **Step 5 (UAT 및 배포)**: 
  * 로컬 검증 완료 후 Vercel 실서버 최종 배포 및 15번 보안_보안관 정밀 스캔 통과 확인.
