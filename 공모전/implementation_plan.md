# [출판친구] 에이전트 통제실 이원화 프로세스 및 이지퍼비터(ezpubitor) 자동화 구현 계획서

본 계획서는 출판친구 플랫폼의 최고 관리자 통제 영역 보안 격리(이원화) 및 1~8단계 무인 출판 파이프라인 핵심 엔진인 '이지퍼비터(ezpubitor)'의 자율 원가 최적화/조판/검수 자동화 시스템 구축을 목표로 합니다.

## 1. 에이전트 통제실 이원화 프로세스 (Security & Session)

실무 영역(출판사 주문, 정산)과 최고 관리자 관제 센터를 논리적·물리적으로 완전히 격리하여 비즈니스 보안성을 강화합니다.

### 1.1. 관리자 전용 통제실 격리 및 접근 제어 (`control.html`)
*   **물리적 격리:** 실무 메인 코드([index.html](file:///c:/Users/seo%20sang%20won/001.작업파일/004. 출판친구/018. 안티그래비티/index.html))에서 관제용 레거시 마크업 및 스타일을 완벽히 제거하여 경량화하고, 별도의 통제실 페이지([control.html](file:///c:/Users/seo%20sang%20won/001.작업파일/004. 출판친구/018. 안티그래비티/control.html))로 분리합니다.
*   **권한 필터 가드:** 페이지 로드 시 브라우저 세션 정보(`sessionStorage`)를 상시 체크하여 `isLoggedIn === 'true'` 및 `userRole === 'admin'` 이중 조건을 충족하는 경우에만 렌더링을 허용하고, 미충족 시 로그인 페이지로 강제 리다이렉트합니다.
*   **새 탭 세션 유실 결함 차단:** `target="_blank"`로 새 창 이동 시 `sessionStorage`가 격리 소실되던 문제를 양방향 이동 링크의 새 창 열기 속성 제거를 통해 해결하여 재로그인 없는 매끄러운 세션 유지를 실현했습니다.

### 1.2. 자율 자가치유 및 무중단 배포 연동 (9번-10번-11번-12번 체인)
*   **에러 격리 (12번 감시_눈치왕):** 예외 발생 시 에러의 핵심 메타데이터(`data.detail`, 200자 제한)만 격리 추출하여 디스코드 알림을 송출하고 사용자 브라우저에는 AI 안심 케어 패널을 노출합니다.
*   **보안 검증 (12번 실시간 AI 보안관):** 악성 인젝션 패킷 정적/시맨틱 1차 필터링 및 10번이 수정한 코드에 포함된 민감 API Key 노출 여부를 30초 단위로 감시/마스킹합니다.
*   **자가치유 & 원클릭 배포 (10번 개발팀장 & 11번 배포관리자):** 10번이 Gemini API로 생성한 핫픽스 PR을 대표님이 모바일(디스코드 훅) 터치 한 번으로 승인하면, 11번이 GitHub squash merge를 자동 격발하여 Vercel 프로덕션에 무중단 배포를 완료합니다.

## 2. 이지퍼비터(ezpubitor) 자율 원가 최적화 엔진

소량 제작 환경에서 실물 인쇄소 장비의 물리적 제약(낱장 트레이 vs 연속지 롤)에 따른 단가 변곡점을 인식하여, AI가 직접 판형 조정을 통한 원가 절감을 역제안하는 자율형 컨설팅 모듈입니다.

### 2.1. 인쇄 장비별 원가 데이터셋 매핑
*   **디지털 낱장 (Sheet-fed - 315x467mm 용지 트레이):**
    *   A5 이하 (148x210mm) 규격: 1장에 4페이지(4판거리) 인쇄가 가능하여 면당 **8원** 부과.
    *   신국판 (152x225mm) 규격: 1장에 2페이지(2판거리)만 인쇄 가능하여 면당 **12원** 부과 (50% 원가 급증).
*   **디지털 연속지 (Web-fed - 440mm 폭 롤):**
    *   신국판 이하 규격까지 롤 종이에 2판을 연달아 걸어 대량 생산하므로 면당 **8원** 요율 유지.
    *   연속지 가동 최소 주문 임계치: **50부** 이상으로 제한.

### 2.2. 자율 원가 최적화 컨설팅 로직 (3번 수익성 분석 & 16번 지휘_판다)
*   주문 부수가 50부 미만(디지털 낱장 인쇄 강제)이면서 업로드된 도서 사양이 신국판인 주문 건을 실시간 감지합니다.
*   3번 에이전트는 신국판(면당 12원) 대비 A5(면당 8원)로 판형을 변경했을 때의 예상 절감액 및 순이익률 향상 폭을 시뮬레이션합니다.
*   16번 지휘_판다는 비침입형 슬라이드 패널(AI 헬퍼) 내부에 **"A5 판형 조정을 통한 원가 33.3% 절감 제안 카드"**를 동적으로 생성하여 노출합니다. 대표자 또는 출판사가 이를 승인할 경우, 4번 조판_조판이 에이전트가 A5 규격으로 자동 리플로우 조판을 자율 처리합니다.

## 3. 폰트/표지 무경로 자동 변환 및 스티칭 아키텍처

출판사의 수정 공수를 최소화하고 폰트 라이선스 문제를 원천 해결하기 위해 백엔드 PDF 제어 엔진이 수행하는 물리적 재구성 기법입니다.

### 3.1. 폰트 유실 및 라이선스 제로 가드레일 (Font Embedding)
*   **벡터 비례 축소 (Vector Scaling):** 출판사가 인쇄용으로 업로드한 신국판 PDF는 이미 모든 폰트 정보가 파일 내부에 포함(임베디드)되어 있습니다.
*   우리는 텍스트 문자를 직접 고쳐 쓰는 것이 아니라, PDF 페이지 전체를 좌표 변환 매트릭스(`scale: 0.93`)를 활용해 가로세로 비율을 유지하며 축소하는 것이므로 **서버에 해당 폰트 파일이 전혀 없어도 폰트 유실이나 깨짐 없이 완벽하게 A5 규격으로 출력**됩니다.
*   **표지 아웃라인 규격화:** 표지의 경우 텍스트를 도형 경로(Vector Path)로 깨부수는 아웃라인 처리를 전제로 하므로 저작권 분쟁 및 폰트 미스매칭이 발생하지 않습니다.
*   **일반 텍스트 원고 조판 시:** 4번 조판사에 플랫폼 자체 상업용 무료 라이선스 폰트(KoPub바탕체, 나눔서체 등)를 기본 바인딩하여 안전을 수호합니다.

### 3.2. 표지 3단 슬라이싱 & 스티칭 (Slicing & Stitching)
*   신국판 표지 PDF가 유입되면, 백엔드 PDF 엔진(`pdf-lib`)이 이를 [앞표지 / 뒷표지 / 책등(세네카)] 영역으로 3단 슬라이싱 분할합니다.
*   앞표지와 뒷표지는 A5 규격(세로 210mm, 가로 148mm)에 맞춰 비례 축소하고, 책등은 조판 결과에 따른 정확한 세네카 두께(mm) 간격으로 벌립니다.
*   이 세 영역을 결합하여 하나의 완벽한 **A5 표지 스프레드 PDF로 백엔드에서 자동 재조립(Stitching)**하여 표지 찌그러짐을 방지합니다.

## 4. 비대면 3D 가상 도서 뷰어 검수 및 무비용 인프라

사용자 편의성을 극대화하고 플랫폼 유지비용을 0원으로 수렴시키기 위한 프론트엔드 및 인프라 설계 규칙입니다.

### 4.1. 브라우저 기반 3D 도서 가상 시뮬레이터 (3D Book Flip Viewer)
*   출판사가 직접 복잡한 수정본을 확인할 수 있도록, A5 변환이 끝난 내지/표지 PDF를 기반으로 3D 책 넘기기 가상 시뮬레이터를 렌더링합니다.
*   오픈소스 3D 라이브러리인 **`Three.js`** 혹은 **CSS3 3D Transform**을 적용하여 플로팅 뷰어를 띄웁니다.
*   모든 3D 연산은 **사용자 기기의 GPU(Client-side)**에서 수행되므로 플랫폼 서버 자원 소모와 트래픽 비용은 **완벽하게 0원**입니다.
*   출판사는 파일을 다시 업로드하는 스트레스 없이 가상 목업을 확인한 뒤 **[최종 최적화 승인]** 버튼 클릭 한 번으로 인쇄 발주를 종결합니다.

### 4.2. 100% 오픈소스 및 무비용 인프라 운영 정책
*   **무비용 개발 라이선스:** 상용 라이선스 계약이 필요한 PDF 변환 SDK 대신, 100% 무료인 MIT 라이선스 자바스크립트 엔진(`pdf-lib`)을 활용하여 조판/편집 모듈을 구동합니다.
*   **인프라 무료 티어 극대화:** 
    *   **Supabase Free Tier:** 데이터베이스 500MB, Storage 1GB 혜택으로 초기 운영비 0원 방어.
    *   **Vercel Hobby Tier:** 호스팅 및 서버리스 쿼리 한도로 API 구동비 0원 방어.
    *   볼륨이 커지더라도 월 $25 수준의 인프라 정액 요금제만으로 운영되며, 이는 도서 판매 마진 몇 권만으로 완벽 상각됩니다.

## 5. 살피미 키워드 로테이션 및 파이프라인 흐름

*   **하드코딩 제거 및 키워드 표준화:** `control.js`에 내장되었던 가상 데이터를 제거하고, 7대 표준 키워드(절판 인문학, 품절 한국소설 등) 로테이션 큐를 자율 순환하여 데이터를 수집합니다.
*   **자동 건너뛰기(Auto-skip) 가드:** 검색 결과 0건 발생 시 실패 원인을 16번 지휘_판다 감사 로그에 남기고 다음 순번으로 즉시 패스하여 파이프라인 정지를 방지하며, 최대 3회 연속 실패 시 폭주 방지를 위해 루프를 안전 차단합니다.
*   **수요 온도 배지 렌더링:** 대출 횟수에 비례한 온도별(🔴 끓는점 90℃↑, 🟠 고온 등) 색상 배지를 실시간 렌더링하여 의사결정을 지원합니다.

## 6. Verification Plan (검증 계획)

### 6.1. 시나리오 및 기능 검증
1.  **세션 격리 테스트:** 일반/출판사 계정 로그인 시 통제실 메뉴가 숨겨지는지, `control.html` 주소 다이렉트 쳐서 들어올 시 차단 및 `index.html` 리다이렉트가 견고히 이루어지는지 검증.
2.  **낱장/연속지 변환 테스트:** 부수를 30부로 지정하고 신국판 책 주문 시, A5 원가 최적화 변환 추천 카드가 AI 헬퍼 패널에 정상 표출되는지 테스트.
3.  **PDF 비례 축소 및 스티칭 테스트:** 업로드한 신국판 PDF 본문과 3단 표지 스프레드가 깨짐 없이 A5 표준 비율 및 책등 두께로 자동 절삭·재조립되는지 검증.
4.  **3D 뷰어 구동성 테스트:** 모바일 및 데스크톱 브라우저 환경에서 CPU/메모리 오버헤드 없이 3D 책 넘기기 모션이 매끄럽고 가볍게 렌더링되는지 리소스 모니터링 수행.
