# [출판친구] 스토어 정체성 강화 (독자 참여형 복간 펀딩 플랫폼) 개편 실행계획서

본 계획서는 출판친구 스토어를 단순 도서 홍보관에서 **'독자 참여형 절판도서 복간 펀딩 플랫폼'**으로 고도화하고, 독자 동기부여를 위한 마일리지 제공 및 가변 데이터 인쇄(VDP) 연동 서포터 증서 발행, B2B 타겟 광고 비즈니스 모델로의 확장을 위한 인프라 구축을 목표로 합니다.

---

## User Review Required

> [!IMPORTANT]
> **1. 가변 데이터 인쇄(VDP) 증서 실물 배송 연동**
> * 복간 도서의 펀딩 참여 독자에게 이름이 개별적으로 인쇄된 '실물 복간 보증서(책갈피)'를 발송하기 위해, 백엔드 `pdf-lib`로 개별 PDF를 동적 생성하는 가변 조판(VDP) 연동 방식을 채택합니다.
>
> **2. 일반회원(B2C) 및 기업회원(B2B) 로그인 물리적 분리**
> * 스토어 메인 상단에 `[일반회원 로그인/가입]` 버튼과 `[기업회원 로그인]` 버튼을 이원화 배치하여 로그인 동선을 이원화합니다.
>
> **3. 투표 무결성 검증 (HMAC-SHA256)**
> * 투표 및 복간 요청의 위조/조작 방지를 위해, `[회원 ID + 도서 ID + 타임스탬프]`를 해싱 및 서명 처리하여 DB에 적재함으로써 조작 없는 민주적 플랫폼임을 보장합니다.
>
> **4. 신규 제안 실시간 살피미 에이전트 연동**
> * 사용자가 신규 도서 제안 시, 즉각 1번 살피미가 가동되어 국립중앙도서관 API에서 도서 정보를 수집하고 16번 판다의 대시보드(통제실)에 실시간으로 알림을 발생시키도록 설계하여 살아있는 자율 멀티 에이전트 시스템을 시각화합니다.
>
> **5. 보안 락(RLS) 정책 전환**
> * 공모전 시연 단계(6/20까지)의 안정성을 보장하기 위해 RLS 정책을 일단 `USING (true)`로 느슨하게 운영하나, 향후 실제 서비스 오픈(Production) 시점에는 `auth.uid() = user_id` 세션 매칭 기반의 엄격한 보안 통제로 강제로 전환해야 합니다.
>
> **6. 로그 서명의 트리거(Trigger) 자동화 고도화**
> * 현재 서명 생성은 API(애플리케이션) 단에서 처리하지만, 추후 상용화 단계에서는 DB 트리거(`CREATE TRIGGER`)를 설정하여 `supporter_logs` 테이블에 인서트가 발생할 때 DB 엔진이 내부에서 서버 비밀키를 직접 참조해 HMAC-SHA256 디지털 서명을 원천 강제 생성하도록 구조를 개선할 예정입니다.
>
> **7. 7번 그림이 삽화 생성 & 11번 알리미 실물 '마녀' 숏폼/카드뉴스 제작 연동 고도화**
> * B2B ERP에서 실증 원고인 '마녀' PDF 업로드 ➔ 가조판 승인 시, 통제실(`control.html`)에서 7번 그림이가 소설 속의 핵심 씬을 묘사하여 삽화를 생성하고, 11번 알리미가 카드뉴스 및 숏폼 비디오(오디오 EQ 이퀄라이저 파형 결합) 에셋을 제작하는 실시간 워크 로그 시뮬레이션을 구현합니다.
> * 시연 안정성과 발표 딜레이(타임아웃) 예방을 위해, 통제실 마케팅 관제 패널에 미리 고해상도 삽화와 폰트를 얹어 준비 완료된 '마녀' 전용의 카드뉴스 5장 캐러셀 및 홍보용 숏폼 비디오 재생창이 2초 내에 고속 로드되는 UAT Mocking 가드레일을 이식합니다.

## Verification Plan

### Manual Verification
1. **회원가입 및 투표 마일리지 적립 테스트:**
   * 일반회원 로그인 ➔ 복간 대기 도서 검색 ➔ 투표 ➔ 득표수 증가, 마일리지 100P 적립, DB 내 위조 불가능한 HMAC 해시 서명(digital_signature) 기록 확인.
2. **실시간 살피미 및 판다 연동 테스트:**

3. **가변 복간 보증서 생성 테스트:**
   * 펀딩 성공 도서 영역에서 가변 증서 발행 시뮬레이션 클릭 ➔ 브라우저에서 후원자 이름이 임베딩된 고해상도 PDF 증서가 깨짐 없이 1초 이내에 자동 조판 및 다운로드되는지 검증.
