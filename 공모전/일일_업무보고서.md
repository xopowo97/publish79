- **3차 진압 (방어 패치 v3 완결판)**: `SUPABASE_URL` 미포함 프로토콜로 인한 Node.js 네이티브 fetch 엔진 Fatal(`TypeError: Invalid URL`) 완전 봉쇄.
  - 4중 방어 가드: ENV 존재 → `startsWith('https://')` → `new URL()` 파싱 → status 코드 전용 확인
  - 전역 최외곽 `try-catch` 이중 래핑: 내부 `_handle()` 어떤 Fatal에도 외벽이 `res.status(200).send()`로 마감 → Vercel 500 구조적 불가능 달성

### 1. 한글 깨짐(인코딩) 해결 및 Supabase 원격 데이터 복구
- **배경**: 상상아카데미 계정으로 로그인 후 [현재 적용 단가표 확인하기]를 누르면 VIP 등급 단가표가 정상 연동되지 않는 오류 발생.
- **원인 분석**: 빌드 도중 한글이 깨진 상태로 배포되어, 파트너사 정보 저장 시 DB `partners` 테이블 내 상상아카데미(`sangsang`)의 `grade` 컬럼에 깨진 텍스트 `"VIP"`가 저장됨. 이로 인해 단가 키(`VIP등급`) 매칭 실패 유발.
- **해결책**:
  - 로컬 `script.js` 및 `index.html` 의 인코딩을 **UTF-8 with BOM**으로 정밀 변환하여 Vercel 배포 환경에서 한글 깨짐을 근본 방지.
  - Supabase `partners` 테이블 원격 제어를 통해 상상아카데미(`sangsang`) 레코드의 `name`을 `'상상아카데미'`, `grade`를 `'VIP등급'`으로 정상 복구 완료(Hex 바이트 검증 완료).
  - 단가 데이터가 든 `master_config` 테이블의 `config` JSON은 실제 디스크상에 한글이 깨지지 않고 완벽 보존되어 있음을 바이트 분석으로 교차 검증 완료.

### 2. 연속지 슬라이딩 단가 전체 구간(5구간) 노출 오류 수정
- **배경**: 적용 단가표의 연속지(롤) 단가 구간이 VIP 등급의 경우 5구간으로 설정되어 있으나 3구간만 노출되어 대표님이 착각하시는 오류 유발.
- **원인**: `script.js` 4956라인 렌더링 로직에서 `(rs.ivs || []).slice(0, 3)`으로 상위 3개 구간만 노출하도록 하드코딩되어 있었음.
- **해결책**: `.slice(0, 3)`을 완전히 제거하여 DB에 보관된 모든 단가 구간(5개 이상)이 동적으로 제한 없이 렌더링되도록 정밀 타격 수정 완료.
