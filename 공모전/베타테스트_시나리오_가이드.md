# 🚀 [출판친구] 실증 도서 1종 기반 베타테스트 시나리오 가이드 계획서 (B2C-B2B 라이브 완공본)

본 문서는 출판친구 플랫폼의 상용화 가능성을 입증하기 위해, **실제 출판사의 동의를 얻은 실증 도서 1종의 데이터**를 활용하여 시스템의 전 프로세스(제안부터 생산대기까지)를 모의 검증하는 공식 시나리오 계획서입니다. 최종 제출한 기획서 문서의 그림 1~10 및 17대 자율협업 에이전트 아키텍처와 100% 동기화하며, 실무 파일 피드백 거버넌스 및 B2C 스토어 - B2B 통제실 간의 **"실시간 복간 펀딩 라이브 연동"** 설계를 반영했습니다.

*   **대상 실증 도서**: [출판사명 기재 예정] - [실증 도서명 기재 예정]
*   **테스트 환경**: 출판친구 B2B ERP 포털, 에이전트 통제실, B2C 펀딩 스토어
*   **제한 및 우회 사항 (시연 안정성 극대화)**: 
    *   **B2B ERP 최종 주문 거버넌스**: 펀딩이 100% 달성되어 일괄 예약 결제가 격발(Toss/PortOne 빌링)된 후, 시스템이 인쇄소에 자동으로 직접 발주를 넣지 않습니다. 출판사가 최종 원고 검수 및 일정을 조율할 수 있는 제어권을 보장하기 위해, 출판사 회원이 직접 ERP에 접속하여 최종 **[도서 제작 등록 및 발주]를 수동 클릭**한 후 인쇄소 대시보드(칸반)로 주문이 정식 인계되는 안전한 실무 흐름을 입증합니다.
    *   **고속 UAT 연출 (비용 0원 및 지연 예방)**: 발표 심사 현장에서 대용량 PDF를 실시간 업로드하고 AI API를 처음부터 연산할 경우 서버 타임아웃이나 인터넷 속도 저하 등의 돌발 사고가 날 수 있습니다. 따라서, 업로드 및 변환 클릭 즉시 **백그라운드에서 미리 준비 완료된 실증 도서(A5 국판)의 완성 표지 이미지와 다운로드용 조판 PDF 데이터가 2초 내 고속 로딩되도록 모방 UAT(Mocking)**를 탑재하여 화려하고 안정적인 고속 시연을 달성합니다.

## 3. 대통합 5단계 상세 시나리오 전개 계획

### 01단계: B2B 영업 제안 및 B2B 출판사 제안서 조회/승인
*   **목표**: 공공 API에서 절판 도서를 탐지하여 타겟 출판사명으로 원가 절감 제안서를 자동 발행하고, 출판사가 B2B ERP CS창에서 이를 승인하는 전 과정을 검증.
*   **액션**:
    1. **[에이전트 파이프라인 가동]**: 대표자가 어드민 통제실에서 [🚀 파이프라인 실행]을 클릭하면 `1번 살피미`와 `2번 다듬이`가 공공 API(도서관 정보나루)를 실시간 수집하여 절판 후보를 감지하고, `3번 계산이`가 신국판 대비 A5 전환 시의 원가 절감율(33.3%)을 연쇄 연산함.
    2. **[영업이 제안서 생성]**: 계산이 끝난 후 `10번 영업이`가 해당 **[대상 출판사명]**과 **[대상 도서명]**을 하이라이트 표기한 최적화 제안서 카드를 자동 생성함.
    3. **[출판사 ERP 확인 및 상담이 연동]**: B2B 파트너 출판사 계정으로 ERP 포털에 접속한 뒤 우측 하단의 **`17번 상담이(고객응대 CS)`** 챗봇 창을 열어 "제안서 확인"을 입력.
    4. 상담이 챗봇 창에 팝업된 **[A5 국판 최적화 제안서 카드]**의 내용을 확인하고 **[A5 판형 최적화 승인]** 버튼을 직접 클릭함.

### 02단계: 4번 조판이 & 8번 이지퍼비터 POD 자동 가조판 및 최종 펀딩 요청
*   **목표**: 출판사가 파일을 업로드하여 AI 가조판 결과를 시각적으로 검수하고 B2C 펀딩 개설을 최종 요청.
*   **액션**:
    1. **[원고 파일 업로드]**: 1단계 승인 즉시 `17번 상담이` 챗봇창 내에 활성화된 파일 업로드 UI를 통해 출판사 담당자가 표지/내지 PDF 파일을 업로드함.
    2. **[가조판 및 커버 정밀 보정 (고속 모방 UAT)]**: 
        *   업로드 즉시 `4번 조판이`와 `8번 이지퍼비터 POD`가 가동되는 비주얼 연출 시작.
        *   네트워크 및 API 지연을 우회하여, 미리 준비된 **실증 도서의 최종 A5 가조판 PDF 데이터**와 책등(Spine mm) 두께가 정렬되어 완성된 **커버 프리뷰 이미지**를 2초 내에 고속 로드하여 챗봇 패널에 제공함.
    3. **[가조판 PDF 검수 및 펀딩 개설]**: 출판사 담당자가 가조판 프리뷰 이미지와 다운로드된 A5 PDF 결과물을 확인한 뒤, 만족 시 `17번 상담이` 말풍선 내의 **[최종 펀딩 개설 요청]** 버튼을 클릭함. 즉시 B2C 펀딩 스토어에 해당 도서 페이지가 실시간 활성화됨.

### 03단계: 11번 알리미 마케팅 가동 및 유튜브 API 실시간 배포
*   **목표**: 펀딩과 동시에 AI 마케팅 에셋을 제작하고 외부 채널에 실제 배포하여 사전 마케팅 공급.
*   **액션**:
    1. **[본문 파싱 및 마케팅 에셋 자동 빌드]**: 펀딩 개설과 함께 **`11번 알리미`**가 최종 확정 텍스트 본문을 정밀 파싱하여 시놉시스에 어울리는 홍보 카드뉴스(5장 캐러셀) 및 성우 TTS 나레이션 + 실시간 댄싱 오디오 파형(EQ bars)이 결합된 9:16 숏폼 비디오 시연 모듈을 즉시 완공함.
    2. **[진짜 API 배포 UAT]**: 대표자가 통제실 화면에서 [SNS 마케팅 최종 배포 승인] 버튼을 클릭하면, 백엔드에서 **YouTube Data API v3**가 연동되어 MP4 숏폼 비디오 파일을 실제 유튜브 공식 테스트 채널에 실시간으로 업로드 처리.
    3. 화면에 출력된 유튜브 쇼츠 라이브 링크를 클릭해, 유튜브 채널에 방금 올라간 영상을 그 자리에서 검증.
    4. **[넥스트 마케팅 확장 로드맵]**: 향후 인스타그램 Reels/TikTok API를 통한 숏폼 동시 업로드 기능 및 네이버 블로그 API 연동 기반의 자율 포스팅 배포망 확장 비전을 제시.

### 04단계: B2C 스토어 펀딩 100% 임계점 돌파 및 실시간 라이브 연동 (1클릭 UAT)
*   **목표**: 펀딩 목표 100% 달성 순간의 실시간 대시보드 및 스토어 라이브 현황판 동시 갱신 입증.
*   **액션**:
    1. B2C 펀딩 스토어의 해당 도서 펀딩 페이지를 띄움. 현재 달성률은 목표 50부 중 **49부**.
    2. 대표님이 스토어 화면에서 마지막 **50번째 펀딩 구매 클릭**을 수동으로 실행.
    3. 펀딩이 즉각 100% 달성되며, 예약 등록된 독자들의 빌링키 결제가 일괄 성공(Toss/PortOne API UAT 격발).
    4. **[B2C-B2B 실시간 듀얼 연동]**:
        *   **B2B 어드민 통제실(`control.html`)**: `누적 복간 도서 수`, `누적 매출액`, `이산화탄소 절감 및 소나무 누적 상쇄 카운터(7,500그루 상당)` 지표들이 실시간(Supabase Realtime) 반응하여 수치가 올라가는 비주얼 확인.
        *   **B2C 독자 스토어(`index.html`)**: 스토어 상단 위젯인 **[실시간 복간 펀딩 라이브 현황판]**의 총 펀딩 금액, 진행 펀딩 수, 누적 참여 독자 수 수치가 새로고침 없이 즉각 상승 연동되어 독자들에게 전문 펀딩 플랫폼으로서의 강력한 브랜드 신뢰감을 전파함.

### 05단계: B2B ERP 최종 발주 및 인쇄소 칸반 대기 확인 (Full-Loop 완성)
*   **목표**: 펀딩 완료된 주문을 출판사가 검수 후 최종 발주하여 인쇄소 칸반 보드에 안착시킴으로써 전 과정 연동 검증.
*   **액션**:
    1. 펀딩이 성공하여 일괄 예약 결제가 격발 완료되었음을 확인한 출판사 회원이 출판친구 B2B ERP에 직접 로그인.
    2. 출판사 권한으로 **[최종 도서 제작 등록 및 발주]**를 직접 클릭하여 발주를 최종 컨펌 (출판사 최종 파일 제어 및 제작 승인 거버넌스 락 검증).
    3. 인쇄소 마스터 계정으로 전환 로그인하여 B2B ERP 페이지의 [생산진행관리] 칸반 보드로 접속.
    4. 칸반 보드의 `[접수 대기]` 열에, 방금 발주된 실증 도서 주문 카드가 성공적으로 등록되어 있는 것을 최종 확인.
    5. 카드를 클릭해 자동 렌더링된 엑셀 작업지시서와 원본 PDF 파일 다운로드 경로가 정상 제공되는지 검토.

## 🖥️ 에이전트 통제실 실시간 어드민 관제
* 상기 1단계부터 5단계까지 출판사와 에이전트, 인쇄소 간에 오고 가는 모든 파일 전송 상태, 의사결정 트래픽, API 응답 상태, 눈치왕의 실시간 감시 상태가 **어드민 통제실(`control.html`) 화면의 실시간 트래픽 로그와 에이전트 LED 불빛 점멸(대기-회색/작동-주황/완료-초록)**을 통해 심사위원단에 한눈에 시각적으로 완벽하게 관제됨을 보장합니다.
