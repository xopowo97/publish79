# [9번 기술행정지원] Gemini API 기반 자율 에러 분석 및 디스코드 연동 실행 계획

본 계획은 제미나이 웹의 수동 'Gem' 활용 방식을 넘어, **Google Gemini API를 소스 코드 단에 직접 이식**하여 대표님의 개입이 전혀 필요 없는 백엔드 자율형 파이프라인의 핵심인 12번 감시_눈치왕 에이전트를 구현하는 상세 실행 계획입니다.

---

## 1. 대표님 검토 요구 사항 (보안 및 API 연동)

> [!IMPORTANT]
> **1. Google Gemini API 키 설정 (`GEMINI_API_KEY`)**
> - 백엔드(`/api/send-error.js`)에서 Gemini API를 호출하기 위해 구글 API 키가 필요합니다.
> - 대표님은 구글 AI Studio에서 발급받은 API 키를 Vercel 배포 대시보드 환경변수(Environment Variables)에 `GEMINI_API_KEY`라는 이름으로 등록해주셔야 합니다.
> 
> **2. 외부 NPM 패키지 미설치 (REST API 다이렉트 통신)**
> - Vercel 서버리스 배포 시 종속성 오류를 방지하고 속도를 최적화하기 위해, 별도의 `@google/generative-ai` npm 패키지를 설치하지 않고 Node.js 표준 `fetch` 함수를 사용하여 REST API 방식으로 구글 Gemini API를 다이렉트 호출합니다.
> 
> **3. 안전한 Fallback(예외 처리)**
> - `GEMINI_API_KEY` 환경변수가 아직 세팅되지 않았거나 Gemini API 서버 장애가 발생하더라도, 단순 에러 모니터링 알림은 누락 없이 디스코드로 즉시 전송될 수 있도록 견고한 예외 처리 코드를 포함합니다.

## 2. 변경 대상 파일 및 코드 설명

- `process.env.GEMINI_API_KEY`가 존재할 경우, 구글 Gemini API의 REST 엔드포인트(`https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-flash:generateContent?key=...`)로 통신을 전송합니다.

- 만약 API 키가 누락되었거나 통신 오류가 날 경우에는, AI 분석 파트만 생략한 채 기존 디스코드 알림을 그대로 송출하도록 예외 처리합니다.

## 3. 검증 계획 (테스트 방법)

### 수동 테스트
1. 로컬 개발 환경의 Vercel 서버리스 구동 상태(또는 `.env` 환경 변수 구성)에서 API 기능 검증.
2. 메인 페이지 하단의 **AI Helper** 패널에서 **DEMO** 버튼을 클릭하여 강제 에러 발생 유도.
3. 대표님의 디스코드 채널에 수신된 알림 메시지에 **"🤖 9번 기술행정지원 실장 AI 진단 리포트"**가 상세하고 친절한 문구로 포함되어 도착하는지 확인.
   * *Fallback 검증*: Gemini API 키 미설정 시에는 AI 리포트 없이 기존 형태의 에러 알림이 정상적으로 수신되는지 확인.
