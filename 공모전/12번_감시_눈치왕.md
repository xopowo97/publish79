## 3. 세부 구현 로드맵

### ✅ Step 1: 모바일 알림 연동 (Slack/Discord Webhook)
*   대표님으로부터 수신할 알림 웹훅 주소를 전달받아 소스코드 또는 Vercel 환경 변수에 등록합니다.
*   백엔드 API에서 해당 웹훅으로 테스트 메시지를 성공적으로 전송할 수 있는지 검증합니다.

### ✅ Step 2: 실시간 에러 감지 미들웨어 구축
*   웹 플랫폼의 프론트엔드(React/HTML)와 백엔드(Serverless/Supabase) 전체에 전역 에러 캐처(Global Error Catcher)를 설계합니다.
*   에러 발생 시 `Stack Trace`, `발생 시간`, `사용자 세션 정보`를 취합하여 로그 데이터 구조를 정형화합니다. (완료 - script.js에 이벤트 리스너 탑재)

### ⏳ Step 3: 에러 로그 DB 적재 및 백엔드 Gemini API 자율 체이닝 연계 (진행 예정)
*   수집된 에러 로그를 Supabase `system_errors` 테이블에 자동으로 insert합니다.
*   (아키텍처 전격 전환) 단순한 DB 적재에 그치지 않고, 에러 로그 삽입 즉시 백엔드에서 **Google Gemini API**를 자동 트리거하여 9번(에러 분석) ➡️ 12번(보안 검증) ➡️ 10번(수정 코드 생성)으로 이어지는 에이전트 백엔드 체인을 완전 자율 구동되게 개발합니다.

### ✅ Step 4: 사용자 브라우저 케어 UI 연동
*   에러 발생 시 화면이 멈추지 않고, AI 에이전트 팝업창이 뜨며 *"에러를 감지하고 복구를 시작했으니 잠시만 대기해 주십시오"*라는 안심 안내를 띄우는 프론트엔드 연동을 마쳤습니다. (완료)

## 5. API 기반 자율형 파이프라인 연동 설계 (Autonomous API Pipeline Design)
*   **연동 주체**: Vercel API / Node.js 백엔드 환경에서 `google/generative-ai` 라이브러리를 활용하여 직접 호출.
*   **파이프라인 트리거**: 프론트엔드에서 수집된 에러가 서버리스 라우트(또는 Supabase Webhook)에 도달하는 시점.
*   **데이터 파이프라인**: 
    1.  `Raw Error Data` ➡️ **Gemini API (9번 페르소나 적용)** 호출 ➡️ `분석 보고서 (JSON)` 출력.
    2.  `분석 보고서 (JSON)` ➡️ **Gemini API (12번 페르소나 적용)** 호출 ➡️ `보안 오염 검증 결과` 출력.
    3.  `검증 결과` + `대상 소스 코드` ➡️ **Gemini API (10번 페르소나 적용)** 호출 ➡️ `수정 패치 코드` 생성.
    4.  `수정 패치 코드` ➡️ **11번 배포 관리자** ➡️ 대표님 모바일 원클릭 승인 요청 발송.
*   **주의 사항**: 이 문서는 12번 감시_눈치왕 에이전트를 Gemini API로 빌딩할 때 삽입할 System Instruction 프롬프트의 기준서 역할을 하며, 어떠한 수동 조작 프로세스도 포함하지 않습니다.
