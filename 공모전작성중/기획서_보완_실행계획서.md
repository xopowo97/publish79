1. **심사위원 전용 원스톱 데모 계정 도입**
   * **계정 정보**: ID: `guest@culture.go.kr` / PW: `culture123!` / 권한: `judge`
   * **로그인 흐름**: 로그인 시 기술력을 증명하는 **'에이전트 통제실(control.html)'**로 즉시 진입한 뒤, 실무 엔진 이동 시 **6대 핵심 비즈니스 메뉴만 노출**되도록 제어하여 심사위원의 검증 편의성을 최적화합니다.
2. **기획서 내 시각 자료(그림/QR) 대폭 압축**
   * 기존 11개의 과도한 그림 배치를 **5개의 고임팩트 핵심 그림**으로 압축하여 10페이지 권장 분량을 준수하고 가독성을 높입니다.
   * 실시간 API 호출 로그, 자가 치유 화면 등은 실제 웹 데모 시연으로 대체하여 지면 낭비를 줄입니다.

## 2. 세부 실행 계획 (Action Items)

### [1단계] 웹 플랫폼 데모 시스템 연동 및 로그인 로직 구현
* **작업 대상**: `index.html`, `script.js`, `control.html`
1. **평가위원 계정 인증 추가**: ID `guest@culture.go.kr` 로그인 시 통제실(`control.html`)로 강제 이동 및 세션 역할 `userRole = 'judge'` 부여.
2. **통제실 접근 제어 완화**: `control.html` 보안 스크립트에서 `admin` 외에 `judge` 권한도 렌더링을 허용하도록 수정.
3. **사이드바 메뉴 필터링**: `script.js` 내 `applyRoleVisibility()` 함수에 `judge` 권한 추가. 
   * **노출 메뉴 6개**: 주문하기, 정산 및 주문관리, 파트너사 정보관리, 인쇄소관리, 판매도서관리, 생산진행관리.

### [2단계] 기획서 내 시각 자료(그림) 자산 압축 배치 (총 5개)
기획서 본문 내 그림 배치를 다음과 같이 5개로 단일화 및 압축하여 구성합니다.

* **[그림 1] 플랫폼 이원화 자율 협업 구성도** (2번 항목 배치)
  * 서비스의 전체적인 AI 연동 개념 및 사업부별 흐름도.
* **[그림 2] 인쇄 설비 규격(315x467mm) 기준 신국판(2판) vs A5 국판(4판) 임포지션 비교 다이어그램** (3번 항목 배치)
  * 판형 최적화를 통한 내지 인쇄비 33.3% 절감의 물리적 증빙 도식.
* **[그림 3] 1인 기업 의사결정 자동화를 위한 CEO Clone 학습용 JSON 데이터 로그 스키마** (3번 항목 배치)
  * 의사결정 학습 로그 아카이빙 기술력 증빙.
* **[그림 4] ePub3 멀티미디어 인터랙티브 전자책 뷰어 실행 화면** (3번 항목 배치)
  * 번역/일러스트/오디오 동기화가 컴파일된 최종 결과물 퀄리티 증빙.
* **[그림 5] 출판친구 통합 관제 통제실 및 데모 접속 QR 코드** (4번 항목 배치)
  * 통제실/웹 화면 캡처 및 심사위원이 모바일로 즉시 검증 가능한 **단일 QR 코드**.

### [3단계] 기획서 마크다운(.md) 및 워드(.docx) 텍스트 최종 교정
1. **기타 정보 기재**: 기획서 상단 표의 `제품·서비스 유형 ➡️ 기타` 란에 심사위원 데모 계정 정보 기재.
2. **문체 일괄 수정**: 3, 4, 5, 6번 항목 내 모든 어미를 격식체(`~합니다`, `~입니다`)로 완벽 통일.
3. **그림 설명 최적화**: 본문 내 그림 배치 공간 및 설명을 위 5개 축소안에 맞춰 교정.
