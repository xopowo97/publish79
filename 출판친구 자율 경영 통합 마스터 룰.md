# 출판친구 자율 경영 통합 마스터 룰 (Master_Governance_Rules.md)

> **⚠️ 규칙 알림**: 이 프로젝트의 모든 마크다운(.md) 파일명은 반드시 한글로 작성하여 사장님과 AI 에이전트가 쉽게 인지하도록 합니다.
> **📌 현재 집중 목표**: 현장 영업용 실물 서비스 안정화 + 공모전용 AI 시연 UI 완성 + 전방위 BM 특허 명세서 자산화

> **단일 진실 공급원 (Single Source of Truth)**
> 본 문서는 대표님(사령관)과 저(Antigravity)가 '출판친구' 플랫폼을 대한민국 공공데이터 경진대회 대상 수상 및 원천 BM 특허 출원을 위해 구축한 최상위 거버넌스 헌법입니다. 백엔드(Supabase) 및 프론트엔드(Vercel: publish79.vercel.app) 개발 및 13대 에이전트 통제 시 본 문서의 규칙을 절대로 위배할 수 없습니다.

## 1. 프로젝트 비전 및 목표 (Vision & KPI)
- **비전**: 국가 공공데이터를 결합하여 잠든 지식 자산을 발굴하고, 13대 AI 에이전트 군단을 통해 1인 창업자도 기술·유지보수 인력 없이 완벽히 영속할 수 있는 초효율 D2C 자율형 출판·인쇄 생태계 구축.
- **공모전 타깃**: 제14회 범정부 공공데이터 AI활용 창업경진대회 및 제4회 문화체육관광 인공지능·데이터 활용 공모전 [신기술활용(ADX) 부문] 대상 (6월 20일 최종 제출).
- **특허 출원 타깃**: 특허청 직접 전자출원(나홀로 특허) 시스템을 통한 '비즈니스 모델(BM) 및 소프트웨어 원천 특허' 출원번호 즉시 확보.

## 2. 13대 AI 에이전트 군단 및 최상위 통제실 아키텍처

본 플랫폼은 안티그래비티 프레임워크와 Google Gemini API를 소스코드 단에 직접 주입(Chaining)하여 구동되며, 사람이 텍스트를 복사-붙여넣기 하는 행위를 배제하고 백엔드 스트리밍 방식으로 자율 협업합니다.

💡 레이어별 명확한 직책(Persona) 정의
1.	최상위 오케스트레이터 (Master Orchestrator): [추가 신설] 12개 에이전트의 교착 상태(Deadlock)를 상시 감시하고, 과부하 발생 시 'API 라이트(Lite) 모드' 전환 및 멀티 LLM 페일오버(Failover)를 총괄 제어하는 최상위 마스터 브레인.
2.	1번 딥서치_살피미: 국립중앙도서관 API(납본 메타데이터), 도서관 정보나루 대출 통계, 시중 품절/절판 DB를 실시간 크롤링하여 잠든 롱테일 지식 자산 추적 발굴.
3.	2번 정제_다듬이: 수집된 비정형 도서 데이터의 노이즈를 청소하고 Supabase 표준 DB 규격으로 전처리.
4.	3번 수익성·타당성 분석 팀장: 4번 조판사가 산출한 4대 표준 판형별 물리적 스펙(페이지수, 세네카 두께)을 수령하여, 플랫폼 단가 관리 테이블 및 페이퍼프라이스 API 용지 원가를 교차 연동함으로써 각 옵션별 예상 마진을 정밀 계산하여 대표님께 최고의 의사결정 카드(최적 사양 추천)를 제안함.
5.	4번 VDP_조판사: 도서별 스펙 가변형 자동 조판(Trim & Page Adaptive Typesetting - DM사의 개인화 VDP와 구분됨)을 수행하며, 대표님이 승인한 최종 규격(4대 표준 판형 중 1개)에 맞추어 도련(Bleed 3mm), 여백, 쪽번호, 세네카를 완벽히 반영한 최종 인쇄 표준 PDF/X-4 파일을 컴파일 렌더링함.
6.	5번 멀티모달 비전 에디터: 화질이 저하된 스캔본 PDF를 OCR로 인식하고, 깨진 폰트 및 도련선 등 인쇄 사양 적합성을 실시간 자동 보정 검수.
7.	6번 수석 크리에이티브 디자이너: 도서 트렌드에 최적화된 저작권 프리 북 커버 디자인 및 레이아웃 자율 생성.
8.	7번 수석 B2B 영업대표: 대형 출판사 의사결정자를 설득하기 위한 맞춤형 'AI 오디오 오버뷰 제안서'를 생성하여 영업 파이프라인 가동.
9.	8번 자율 마케팅 앰버서더: 복간 확정 시 카드뉴스 및 소개 문구를 자동 생성하여 SNS 채널 배포 및 예약 독자 풀 형성.
10.	9번 기술행정지원 실장: Vercel/Supabase 상의 시스템 오류(CORS 우회, 인코딩 충돌 등)를 감지하고, 사용자 브라우저 상에 안심 케어 패널(ai-panel) 팝업을 즉각 연동하는 고객 방어 레이어.
11.	10번 자가치유 코딩 에이전트: 9번이 격리한 에러 패킷을 분석하여 소스코드 결함 위치를 식별하고, 컨텍스트 격리(Isolation) 방식으로 정밀 타격 수정 코드(Hot-Fix) 생성 후 GitHub PR 발행.
12.	11번 자동화 배포 관리자: 대표님의 모바일(디스코드 등 웹훅)로 최종 코드 패치를 송출하고, [승인 / 반려] 원클릭 입력에 따라 Vercel Production에 무중단 배포를 최종 실행하는 시스템 방장.
13.	12번 실시간 AI 보안관: 에이전트 간 프롬프트 인젝션 및 데이터 교차 오염을 차단하고, 10번이 수정한 코드에 API Key 등 민감 환경 변수가 유출되지 않았는지 30초 단위 전수 보안 스캔을 수행하는 감시관.

## 3. 핵심 비즈니스 모델 및 팩트 통제 규칙 (Revenue & Margins)
당신은 플랫폼 설계 및 코드 수정 시, 대표님이 확립한 출판·인쇄 현업의 실증 통계와 마진 구조를 철저히 방어해야 합니다.
•	용지 및 POD 유통 공급망의 투트랙 분리 구조:
  * **트랙 1 (B2B 용지 판매 중개)**: 출판사/소형 출판사가 초판 대량 인쇄(예: 1,000부 이상) 시 자체 인쇄소로 직접 오프셋 발주를 하되, 용지는 출판친구를 경유해 선결제(Paper Price API 연동)하여 35% 할인율로 공급받을 수 있는 구조를 제공합니다. (개별 소형 출판사가 인쇄소에서 받는 용지 할인율은 20~25% 수준에 그쳐 플랫폼을 경유한 용지구매가 절대적인 마진 이점을 제공합니다.)
  * **트랙 2 (소량 POD 인쇄 위탁)**: 개별 소량 POD 주문에 대해 플랫폼이 하청 인쇄소와 체결한 페이지당 고정 계약 단가(디지털 낱장 및 디지털 연속지/슬라이딩 방식)를 적용합니다. 산재된 소량 도서 인쇄 물량을 플랫폼이 통합 취합하여 인쇄소 가동률을 보장해 줌으로써 최상의 가공 단가 경쟁력을 확보합니다.
•	3번 - 4번 - CEO 최고의사결정 연동 프로세스:
  * 1단계: **4번 VDP_조판사**가 2번 다듬이의 정제 텍스트를 파싱하여 4대 표준 판형별 예상 페이지 수 및 용지 평량별 책등(세네카 mm) 두께 물리 스펙을 1차 가상 산출하여 3번 팀장에게 전송합니다.
  * 2단계: **3번 수익성·타당성 분석 팀장**이 수령한 물리 스펙을 바탕으로, 하청 인쇄소의 고정 계약 단가표(디지털 낱장 및 디지털 연속지 슬라이딩 단가)를 매핑하여 판형별 제작 단가, 예상 이익률, BEP를 시뮬레이션한 '의사결정 카드'를 CEO 대시보드에 업로드합니다.
  * 3단계: **CEO(대표님)**가 대시보드 상에서 4대 판형 옵션의 마진 비교를 보고 최적 판형 1개를 선택하여 승인합니다.
  * 4단계: **4번 VDP_조판사**가 승인 신호를 수령하고, 해당 판형에 맞춰 도련선(3mm), Gutter 여백, 쪽번호 등을 완벽히 반영한 인쇄 표준 고해상도 PDF/X-4 파일을 최종 컴파일 빌드합니다.
•	10부 안전 재고 및 5부 묶음 발주(Batching) 알고리즘: 성수기(10~13%)와 비수기(4%) 사이에 발생하는 3배 이상의 극심한 계절성 가동률 리스크를 극복하기 위해, 주문 데이터를 가변적으로 묶어 인쇄소 윤전 인프라(D2P)에 자동 토스함으로써 최저 단가(신국판 200p 기준 권당 3,100원)를 철통 방어합니다.
•	출판친구 B2B/B2G 파트너 포털 및 B2B2C API 드롭쉬핑 연동: B2C 일반 판매는 교보/예스24/알라딘 등의 대형 서점 유통망과 API로 연동하여 자동 주문 제작/배송(Dropshipping)하는 B2B2C 모델로 운영하고, 자체 웹 플랫폼은 B2B/B2G(도서관, 지자체, 대학)용 주문 및 신청 관제 게이트웨이이자 파트너사(인쇄소) 관리 백오피스 포털로 강력 고정 운영합니다.
•	4대 표준 판형 고정 정책: AI 조판 오류율 0% 수호 및 3번 에이전트의 원가/마진 자동 산출 정밀화를 위해 플랫폼 유통 규격을 [신국판(152x225), A5국판(148x210), 46배판형(188x257), 국배판(210x297)]의 4대 표준 규격으로 강력 제한하여 운영합니다. 이를 통해 용지 통합 발주 할인율(35%)과 인쇄소 D2P 단가 딜 협상력을 배가시킵니다.

## 4. 🔒 특허청 직접 출원을 위한 자율 기술 자료 적재 규칙
당신은 향후 대표님이 변리사 비용 없이 단돈 몇만 원으로 특허청 '특허로' 사이트에 직접 제출할 수 있도록 모든 에이전트 개발 프로세스를 완벽한 공문서 양식으로 누적해야 합니다.
•	자동 문서화 의무: 코드를 패치하거나 새로운 프론트엔드/백엔드 기능을 빌드할 때마다, 특허 명세서 서식에 맞춘 배경기술, 해결 과제, 실시 예, 그리고 에이전트 간 데이터 트래픽의 '텍스트 기반 도면 시퀀스(Sequence Flow) 로그'를 특허청_제출용_명세서_자료.md 파일에 자율적으로 즉시 업데이트하십시오.
•	증명 아키텍처 축적: 12번 보안관이 수행하는 국정원 규격 준수 보안 필터링 데이터와, 10부 단위 버퍼 제어를 통한 3,100원 원가 통제 수학적 시뮬레이션 데이터를 숫자로 정밀하게 기록해 두어야 합니다.

## 5. 📊 [2번 다듬이] 데이터 정제 필터링 정책 및 [자율경영 관제 센터] 대시보드 설계 규칙

본 마스터 룰 하에 개발되는 데이터 정제 모듈 및 통합 대시보드는 다음의 보안·가독성·확장성 아키텍처 규칙을 절대적으로 준수해야 합니다.

### 5.1. 다듬이 에이전트 데이터 정제 필터링 정책 (Filtering Policy)
* **데이터 신뢰도(Data Credibility) 검증 규칙**:
  * **무결성 스캔**: 수집된 도서 서지 정보의 ISBN-13 체크섬 유효성을 즉시 확인하고 누락된 메타데이터(도서명, 저자, 출판사, 발행일)가 있는 불완전 레코드는 필터링 제거합니다.
  * **저작권 분류**: 저작권 만료(사후 70년 경과) 도서를 파악하여 '공공 도메인 복간 후보군'으로 자동 마킹하고, 저작권 만료 도서 및 저작권 정보가 확인되는 서지 정보를 우선순위 분류합니다.
  * **노이즈 정제**: 크롤링이나 비정형 텍스트 내 포함된 HTML 태그 및 UTF-8 BOM 인코딩 깨짐을 강제 교정하여 데이터베이스를 청소합니다.
* **사용자 만족도(User Satisfaction) 및 복간 수요 검증 규칙**:
  * **대출 빈도 매핑**: 도서관 정보나루 대출 통계 데이터를 매핑하여 절판 상태에도 지속적 대출이 일어나는 수요 강도를 정량 분석합니다.
  * **시장 희소성 교차 검증**: 주요 온라인 서점 API를 통해 실제 '품절' 및 '절판' 여부를 실시간으로 대조합니다.
  * **중고 프리미엄 추적**: 정가 대비 중고 거래 가격 상승 비율을 추적하여 소장 가치가 높은 도서를 발굴합니다.
  * **조합용 마스터 키워드**: 복간 타당성 탐색 시 `[절판] + [대출 급증]`, `[품절] + [중고 프리미엄]`, `[저작권 만료] + [학술 명저]`와 같은 정교한 키워드 조합으로 에이전트 질의를 정형화합니다.

### 5.2. [자율경영 관제 센터] 대시보드 확장성 & 가독성 설계 규칙 (Dashboard UI/UX)
* **최초 진입 화면 설계 (Landing dashboard)**:
  * 대시보드는 로그인 이후 대표님(관리자)이 가장 먼저 진입하는 **최초 화면**으로 고정하여 시스템의 자율 가동 가시성을 즉시 전달해야 합니다.
* **조직도 드롭다운 기반 무한 확장성 (Agent Registry)**:
  * 에이전트 목록은 HTML 하드코딩을 배제하고, DB 에이전트 메타데이터 테이블을 연동하여 14번~99번 신규 에이전트가 생성되더라도 좌측 계층형 조직도에 자동으로 렌더링되게 설계합니다.
  * 가로 확장(살피미 1~10호 등 동일 에이전트의 복수 스레드 동작) 및 세로 확장(신규 번호 에이전트 추가)이 조직도 드롭다운 하위에 목록 형태로 가독성 있게 펼쳐지도록 구성합니다.
* **불필요한 재무 정보 제거 및 가독성 정돈**:
  * 복잡하거나 예측 불확실한 "순이익", "비용절감액" 등의 위젯은 화면에서 과감히 제거하여 대표님의 시각 피로도를 방지합니다.
  * 대신 **도서 데이터 수집/정제 처리량 통계** 및 **실시간 복간 대상 추천 도서 리스트 TOP 3** 등 비즈니스 본질에 충족하는 데이터 중심의 위젯을 중앙에 배치합니다.
* **단일 코드베이스 반응형 웹 설계 (Responsive Design)**:
  * 데스크톱 환경에서는 **3열 그리드 구조**(`[좌: 조직도] - [중앙: 통계/추천] - [우: 실시간 로그]`)로 웅장하게 레이아웃을 표현합니다.
  * 모바일 환경에서는 미디어 쿼리(CSS Media Query)를 통해 **상하 1열 세로 스크롤 구조**로 자동 정돈되도록 단일 코드로 설계하여 별도의 모바일 앱 재구축 비용 없이 동일한 관제 환경을 지원합니다.
* **13번 오케스트레이터의 최고의사결정 추천 (대화형 에이전트 흐름)**:
  * 파이프라인 완료 시 1~8단계 시뮬레이터 모달이 화면에 **강제 자동 팝업되는 설계를 전면 폐지**합니다.
  * 대신 파이프라인 완료 시 **우측 하단의 AI 헬퍼 패널이 자동으로 활성화(슬라이드업)**되며, 그 내부에 **13번 오케스트레이터의 '선제작비 확보형 예약 판매 제안' 카드**가 동적으로 삽입됩니다.
  * 대표님이 제안 카드 내의 **[🚀 시뮬레이션 가동]** 또는 **[🚀 예약 판매 개설 승인]** 버튼을 직접 탭해야만 시뮬레이터 모달이 가동되도록 설계하여 "비침입형 AI 거버넌스" 모델을 구현합니다.
  * 제안 카드는 **토스페이먼츠/포트원 빌링키 기반 예약 결제(Pre-authorization) 방식**을 탑재하여, 펀딩 성공 시에만 일괄 자동 정산을 승인하고 펀딩 실패 시 예약을 폐기함으로써 환불 수수료 및 수동 취소 리스크를 0%로 통제합니다.
  * **7대 표준 키워드 로테이션 및 자동 건너뛰기(Auto-skip) 가드**:
    - 1번 살피미 가동 시, 키워드 입력 필드가 비어있거나 기본값일 때 `['절판 인문학', '품절 한국소설', '절판 그림책', '절판 철학', '복간 경제 경영', '저작권 만료 명저', '지역 향토 자료']` 순서로 로테이션하며, `localStorage` 기반 상태 보존으로 브라우저 재시작 시에도 영구 연동됩니다.
    - 특정 키워드 수집 결과가 0건일 경우, 13번 오케스트레이터 감사 로그에 실패 기록 후 즉시 다음 키워드로 **자동 순환 건너뛰기(Auto-skip)**를 수행합니다.
    - 무한 루프 방지를 위해 **최대 3회 연속 실패(Max 3 Retries)** 시 안전 차단하여 중단되도록 설계 가레일을 설정합니다.
* **하이브리드 관제실 전환 및 오버레이 슬라이더 (Hybrid Overlay Navigation)**:
  * 실무 화면(정산, 주문서 작성 등)의 가로 폭 찌그러짐을 방지하기 위해, 슬라이드 인/아웃 작동 시 일반 실무 화면 레이아웃을 밀어내지 않고 화면 위에 겹쳐 뜨는 **플로팅 오버레이 드로어(Overlay Drawer) 방식**을 강제 적용합니다.
  * 좌측 메뉴 최상단에 **`[🤖 에이전트 통제실]`** 메뉴를 배치하여 전체 화면으로 통제실 화면에 진입할 수 있는 메인 루트를 고수하며, 모바일 기기에서는 슬라이딩 제스처 충돌을 방지하기 위해 터치 탭 이동으로 일원화합니다.
* **맥락 인지형 비침입형 광학 펄스 알림 (Context-aware Glowing Pulse FAB)**:
  * 대표님이 실무 작업 중일 때 13번 브레인이 복간 성공률 90% 이상의 고부가가치 도서 발굴 또는 12번 보안관의 보안 침입 차단 등 특이사항을 감지하면, 우측 하단의 **Antigravity AI 동글이 버튼(FAB)**에 부드러운 네온 광학 펄스(Glowing Pulse) 애니메이션을 즉시 가동하여 열람을 정중히 유도합니다.
  * **펄스 프로토콜**:
    - **평상시**: 10초 주기 느린 에메랄드 그린(Green) 숨쉬기 효과 (시스템 정상 가동 증명)
    - **복간 기회 감지 (90% 이상)**: 2초 주기 골드(Gold) 네온 광학 펄스 효과 (기회 포착 알림)
    - **보안 차단 / 긴급 오류**: 1초 주기 레드(Red) 빠른 광학 펄스 효과 (긴급 확인 요청)
  * 버튼 클릭 시 미니 드로어가 슥 열리며 상세 요약 카드가 노출되고, 패널 내 `[통제실 전체화면으로 이동 (⛶)]`을 클릭하면 즉각 대시보드 탭으로 부드럽게 화면이 비행 전환(Transition)됩니다.
* **대표 의사결정 로그 축적 및 분신 에이전트 대비 데이터 설계 (Executive Decision Log & Clone Agent Dataset)**:
  * 13번 오케스트레이터의 최고의사결정 권고에 대하여 대표님이 수행한 [승인 / 반려] 결정을 기록하는 `ceo_decision_logs` 테이블을 Supabase DB에 연동합니다.
  * 저장 시 결과 데이터뿐만 아니라, **의사결정 시점의 정밀 비즈니스 컨텍스트(제안 도서 정보, 대출 건수, 마진율, 원가, 가동률 등)**를 JSON 데이터 포맷으로 묶어 함께 매핑 적재합니다.
  * 특히 반려(Rejection) 조치 시 대표님이 직접 입력하는 세부 반려 사유(rejection_reason)를 텍스트 피처로 수집하여 대표님의 비즈니스 필터링 의도를 데이터베이스에 보존합니다.
  * 본 의사결정 로그 데이터셋은 장기적으로 대표님의 경영 철학과 의사결정 가치관을 학습하여 1~12번 및 신규 N차 에이전트군을 대표님 대신 자동 지휘 및 오케스트레이션할 수 있는 **'대표 분신 에이전트(CEO Clone Agent)'**의 모방 학습(Imitation Learning) 및 파인튜닝(Fine-tuning)을 위한 핵심 원천 자산으로 관리합니다.
* **로그인 상태 연계 및 최고 관리자 전용 권한 제한 (Authentication & Authorization Guard)**:
  * 비로그인 상태(로그인 화면)에서는 AI 동글이(FAB)와 헬퍼창이 외부에 노출되지 않도록 세션 상태(`isLoggedIn === 'true'`)를 체크하여 완전 은닉(`display: none`) 처리합니다.
  * 로그인 성공 후에도, 오직 **최고 관리자(`admin`)** 권한을 가진 사용자에게만 AI 헬퍼(FAB/패널) 및 사이드바 내 '에이전트 통제실' 메뉴를 노출하여 개발용 자가 치유 기능 및 관제 시스템을 안전하게 독점 보호합니다. 일반 출판사나 인쇄소 계정으로 로그인 시에는 해당 요소들을 완전히 숨기고 권한에 적합한 페이지로 리다이렉트되도록 구성합니다.
* **실시간 수집·분석 도서 피드 및 분야별 카테고리 배지 시각화 (Real-time Feed & Category Badges)**:
  * 복간 타당성 점수 기준 미달 등의 이유로 대시보드 TOP 3 데이터가 장시간 갱신되지 않아 발생하는 UX 가동성 피드백을 방지하기 위해, 복간 점수와 무관하게 최근에 수집 및 정제 처리된 도서 8건을 보여주는 '실시간 수집 도서 피드' 위젯을 7일 수요 트렌드 차트 아래에 배치합니다.
  * 수집된 도서의 알라딘 상세 카테고리를 소설, 인문학, 에세이 등 12가지 대분류로 파싱하고, 각 분류에 매칭되는 직관적인 분야별 색상 배지를 부착하여 대표님과 사용자가 수집 결과를 한눈에 식별할 수 있도록 지원합니다.
  * 피드 내에 표시되는 최신 수집 도서 클릭 시에도, 1~8단계 자율 복간 의사결정 시뮬레이터가 정상 구동되도록 API 결합 및 모달 데이터 연동을 연계 설계합니다.
* **5.3. 실전용 인프라 보안 및 데이터 무결성 규격 (Production Database & API Security Rules)**:
  * **시연용 fake 데이터 차단**: 데이터베이스의 스키마 및 적재 영역은 하드코딩된 과거 시연용 INSERT 구문들을 완전히 소거하고, 실제 공공데이터 파이프라인에서 수집·정제된 데이터만을 담아 무결성을 고수합니다.
  - **철통 RLS 및 익명 사용자 차단**: `agent_audit_logs` 및 `reprint_candidates` 등 핵심 통제 테이블은 RLS를 반드시 활성화하고, 익명 사용자의 직접적인 읽기/쓰기 정책을 완전 파기(DROP POLICY)하여 클라이언트 단의 다이렉트 Supabase Anon Key 악용을 원천 차단합니다. 모든 쿼리는 비밀 키(`service_role`)를 품은 Vercel API 서버리스 레이어를 통해 안전하게 실행됩니다.
  - **백엔드 빈도 제한 (Rate Limit) 필터**: 외부 호출 및 DB 부하를 차단하기 위해 백엔드 API 단에 IP별 분당 최대 호출 제한(수집 파이프라인 분당 10회, 추천 후보 조회 분당 20회) 가드를 구현하며, 한도 초과 시 `429 Too Many Requests` 상태 코드를 반환합니다.
  - **UNIQUE 제약 및 멱등성 보정**: `reprint_candidates`의 `isbn` 컬럼은 `UNIQUE` 제약을 강력하게 설정하며, 백엔드 데이터 적재 시 중복 충돌을 방지하기 위해 반드시 `?on_conflict=isbn` 매개변수를 지정하여 덮어쓰기(UPSERT) 처리합니다. 데이터베이스 스키마는 항상 멱등성이 보장되도록 `DO $$` 블록을 활용하여 안전하게 마이그레이션합니다.
  - **실전용 에이전트 실시간 상태 변경 연동**: 파이프라인 가동 API(`/api/pipeline`) 동작 시, 단순히 감사 로그만 남기는 것이 아니라 Supabase의 `agents` 테이블에 있는 개별 에이전트(13번 오케스트레이터, 1번 살피미, 2번 다듬이 등)의 `status` 및 `role`을 실시간으로 업데이트(시작 시 `running` ➔ 완료 시 `success`/`active`)하여 대시보드 조직도의 네온 불빛이 실제 시스템 가동 상황을 실시간으로 반영하도록 구현합니다.
  - **차트 날짜 동적 롤링**: 대시보드의 '7일 수요 트렌드' 선그래프 날짜축 라벨은 접속 당일 날짜를 기점으로 최근 7일을 자동으로 계산하여 동적으로 밀려나오도록(Rolling) 규격화합니다.

## 7. 작업 절차 및 세션 복원 규칙 (Workflow)
1.	새로운 대화 세션이 시작될 때 당신은 가장 먼저 "Master_Governance_Rules.md 읽고 시작하겠습니다"라고 사장님께 보고한 뒤 이 맥락을 복원해야 합니다.
2.	사장님이 코드 수정을 명령하실 때는 오지랖(Hallucination)을 부려 주변 디자인이나 기능을 깨뜨리지 않도록, "지정된 요청사항 이외의 코드 및 디자인은 단 한 줄도 수정하지 않는 정밀 타격(Isolation Edit)" 원칙을 절대 사수하십시오.
3.	기존의 구형 기록 파일인 콘테스트 마스터 브레인과 콘테스트 마스터 드래프트 브레인은 안전한 백업을 위해 신설된 옛날_브레인_백업(old_brain) 폴더로 온전히 이동(Archiving)시키고, 메인 작업 공간은 본 문서만을 단일 진실 공급원으로 삼아 통제합니다.
