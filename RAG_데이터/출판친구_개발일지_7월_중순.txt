# 출판친구 개발 및 설계 일지 (2026년 7월 중순)

## 1. 대시보드 실시간 DB 동기화 및 목업 데이터 제거 (2026-07-13)
* **목적**: 기존 에이전트 통제실 대시보드와 서버리스 API에 녹아있던 시연용 하드코딩(Mock) 데이터 및 '마녀' 1순위 고정 기믹을 완전히 걷어내고, 수파베이스 DB의 실제 대출 수집 통계 및 에셋 생성 지표로의 무결한 동기화 연동을 완수하여 상용화 플랫폼의 데이터 신뢰도를 100% 확보한다.
* **주요 개발 내용**:
  1. **백엔드 API 하드코딩 걷어내기 및 가드레일 장착 (`api/reprint-candidates.js`)**:
     - 기존 API 응답에 1순위로 강제 동적 주입(Inject)되던 '마녀(witchBook)' 도서 코드를 완전히 제거하여, 순수하게 데이터베이스에서 쿼리된 점수순/최신순 데이터만 렌더링되도록 수정.
     - 데이터베이스가 비어있거나(0건) 예기치 못한 통신 장애 발생 시 `undefined` 자바스크립트 크래시를 차단하기 위해, API 리턴 직전 결과를 검증하여 강제로 빈 배열 `[]`로 매핑해주는 Null/Undefined 폴백 가드레일을 장착.
  2. **프런트엔드 대시보드 및 지표 연동 교정 (`control.js`)**:
     - 최초 로드 시 화면을 그리기 위해 가상으로 선언해둔 전역 mock 도서 배열 `_ctrl_candidates`를 빈 배열 `[]`로 초기화하여 최초 진입 시 목업 노출을 완전 소거.
     - 기존에 존재하지 않는 `books` 테이블을 조회하여 404 오류를 뿜던 전체 수집 도서 지표 조회 코드를 실제 원천 테이블인 `reprint_candidates`로 전격 수정.
     - **대표님 피드백 ① 반영 (에셋 테이블 연계)**: 복간 성공 종수(`successKinds`) 계산 시, 단순 펀딩 상태에만 의존하지 않고 실제 배치 엔진([generate_assets_factory.js])에 의해 마케팅 자산 생성이 무결하게 완료된 `book_marketing_assets` 테이블 내의 `status = 'success'` 레코드 건수를 실시간 카운트하도록 쿼리를 보완하여 실적판 수치 매핑을 현실화.
     - **대표님 피드백 ② 반영 (0건 Null 가드)**: 쿼리 결과 비어있음 감지 시, `data.filter` 호출 오류 등을 원천 차단하기 위해 `Array.isArray(data) ? data : []` 2중 Null 폴백 안전 가드를 주입하여 화면 백화(Crash) 방지.

## 2. 대표님 UAT 피드백 수렴 2차 최종 고도화 (2026-07-13 오후)
* **목적**: 브라우저 RLS(보안 규칙) 차단 장벽 및 대시보드 에셋 탭의 하드코딩 로직 누락을 전면 보완하여, 진짜 마케팅 비디오 상영과 실시간 로그 갱신이 오차 없이 100% 동기화 구동되도록 종결한다.
* **주요 개발 내용**:
  1. **RLS 우회 통합 지표/로그 쿼리 설계 (`api/reprint-candidates.js` 백엔드 개조)**:
     - 브라우저 단에서 anon 공개 키로 테이블 직접 count 시 RLS 차단 때문에 `0종` 및 로그 멈춤(--:--:--)이 유발되던 문제를 극복하기 위해, 백엔드 서버 단에서 `SERVICE_ROLE_KEY`를 사용해 RLS를 무력화하고 전체 수집 수, 절판 수, 성공 에셋 수, 파트너 수 및 `agent_audit_logs` 최신 15개 로그를 Promise.all로 병렬 조회하여 JSON 패키지 구조로 리턴하도록 API 인터페이스를 고도화.
     - 특정 ISBN을 넘겨받아 `SERVICE_ROLE_KEY` 권한으로 마케팅 자산을 서버에서 우회 조회해 주는 **개별 에셋 조회용 API 분기(`req.query.isbn`)**를 추가하여 프론트엔드의 RLS 쿼리 차단 완벽 우회 지원.
  2. **프런트엔드 수신 바인딩 및 로그 완전 덮어쓰기 (`control.js` 프론트 개조)**:
     - API 통신 완료 시 넘어오는 `stats` 데이터를 B2B 전략 관제 드로어 패널의 메트릭 카드들에 강제 바인딩하여 0종 멈춤 오류 박멸.
     - DOMContentLoaded 시점에 **5초 간격 정기 인터벌 폴러(`setInterval`)**를 주입하여, 5초 주기로 RLS 우회 API를 쿼리해 통계와 실시간 로그 리스트를 누적/리프레시하고 타임스탬프와 함께 증분 인서트하도록 가동.
     - 드로어가 열릴 때 수파베이스를 직접 조회하여 0종으로 강제 덮어쓰던 쿼리를 소거하고 `window._ctrl_cachedStats` 캐시 객체를 매핑하도록 정정.
  3. **진짜 AI 에셋 페치 및 비디오/표지 렌더링 (`control.js` 에셋 탭 개조)**:
     - 펀딩 가조판 시뮬레이션 완료 및 '마케팅 에셋 UAT 실행' 격발 시, `book_marketing_assets`에서 진짜로 로드된 비디오 파일 주소(`mp4_url`)와 인문학 5장 카드뉴스 정보(`card_news_data`)를 수신하여 플레이어 영역에 즉각 렌더링함.
     - 4번째 Unsplash 이미지 엑박 오류를 해결하기 위해 `photo-1506880018603-83d5b814b5a6` 이미지로 전격 복구 완료하여 브라우저 테스트 무결성 달성.

## 4. 20종 대량 에셋 파이프라인 최종 UAT 연동 마감 승인 완료 (2026-07-14 오후)
* **목적**: Vercel 무료 서버리스 클라우드의 10초 강제 동결(타임아웃) 및 중복 덮어쓰기 비효율 문제를 해결하고, 실제 수집 도서 20종에 대한 마케팅 자산을 100% 완벽하게 적재 성공하여 UAT 최종 승인을 획득한다.
* **주요 개발 내용**:
  1. **Vercel 타임아웃 회피용 프론트엔드 루프 제어 도입 (`control.js`)**:
     - 클라이언트에 완료 응답을 던진 뒤 서버리스 인스턴스가 즉시 동결되는 Vercel의 제약을 극복하고자, 프론트엔드가 백엔드 API를 책 1종씩 동기식(await)으로 연속 호출하게 루프를 제어. 개별 도서는 2초 이내에 안전하게 가동 완료됨.
  2. **진짜 20종 후보 도서 및 에셋 상태 동시 쿼리 (`api/store.js` 신설 분기)**:
     - 대시보드의 TOP 3 임시 배열(`_ctrl_candidates`) 대신, 백엔드로부터 진짜 수집된 20종 도서 리스트를 쿼리해오는 `get-bulk-candidates` API 분기를 신설.
     - 쿼리 시, 이미 에셋 생성이 성공적으로 완료된 도서의 여부(`has_asset: true/false`)를 병렬로 매핑하여 뱉어줌.
  3. **이미 완료된 도서 스마트 스킵 가드레일 (Deduplication Skip)**:
     - `has_asset === true` 인 도서(예: 사랑의 선물, 돈의 방정식)는 백엔드를 부르지 않고 0.01초 만에 즉시 통과시켜 중복 덮어쓰기 낭비를 원천 방어.
  4. **초경량 룰베이스 폴백 가드 주입 (`api/store.js`)**:
     - Gemini API의 간헐적인 지연에 대처하여, `AbortController`를 통해 4.5초 이내에 API가 응답하지 않으면 자동으로 초경량 룰베이스 카피 제작기로 우회하게 설계해 20종 전체가 타임아웃 없이 무결하게 성공(`success`) 상태로 적재 완료되도록 교정.
  5. **숏폼 비디오 시연용 폴백 UAT 안내 보라색 배너 탑재 (`control.js`)**:
     - 클라우드 무료 서버 인프라 한계로 UAT 테스트 시 예전 마녀 유튜브 영상이 재생됨을 오해 없이 설명하기 위해, 스마트폰 프레임 하단에 보라색 시연 안내 라벨을 삽입하여 플랫폼 완성도 제고.

## 5. 노트북 로컬 인프라 에셋 양산 공장화 및 자동 쉬어가기 루프 완공 (2026-07-14 저녁)

     - 노트북에 Node.js 설치 확인 및 윈도우 패키지 관리자(`winget`)를 통한 Gyan.FFmpeg 8.1.2 프로그램 자동 설치 완료.

     - 수집할 도서가 0건일 경우 자율 루프가 스스로 종료되도록 안전장치를 구축하여 CPU와 API의 가성비 및 요금 보호를 최우선으로 수립함.

     - **[최우선]** Supabase 대시보드 Storage 메뉴에 `assets` 라는 명칭의 Public 버킷을 신규 생성 완료하여 Storage 404 Bucket not found 업로드 에러를 해결하기.

## 6. 제미나이 무료 API 일일 한도 초과 오류 규명 및 안전 지연/중복 방지 패치 (2026-07-15)
* **목적**: 7월 14일 구축 완료한 로컬 에셋 양산 공장 가동 시 발생한 `Gemini generateContent failed` 오류의 진짜 원인(일일 무료 쿼터 20회 초과에 따른 Resource Exhausted 429)을 정밀 규명하고, 차후 리셋/유료화 가동 시 API 요율 한도 보호를 위한 4초 안전 지연 설정 및 실패/가동 중인 도서의 중복 뺑뺑이 재호출을 방기하기 위한 1회 가동 세션 내 메모리 셋(failedIsbnsInRun) 필터링 가드를 탑재한다.

  3. **메모리 기반 실패 도서 임시 제외 가드 (Deduplication Guard)**: 1회 가동 세션 동안 에러가 발생한 도서들의 ISBN을 메모리 셋(`failedIsbnsInRun = new Set()`)에 동적으로 등록하고, 루프 리스트 필터링 시 `!failedIsbnsInRun.has(book.isbn)` 조건문을 추가. 이를 통해 동일 실행 중 실패 도서가 무한 루프 뺑뺑이를 돌며 제미나이 API에 불필요한 요청을 가하는 문제를 원천 해결함과 동시에, 재부팅 시에는 정상적으로 복구/재도전이 가능하게 아키텍처를 확보.

     - 사용자가 드롭다운 선택 시 즉시 로컬 캐시 메모리 배열을 정렬 및 필터링해 새로고침 없이 아래 리스트를 실시간으로 다시 그리는 렌더링 핸들러 탑재.
  5. **장르 통계 배지 보드 이식 (`control.html`, `control.js`)**:
     - 관제 센터 패널 하단에 `소설`, `인문학`, `아동`, `저작권 만료`, `사회과학` 5개 장르의 누적 도서 수를 나타내는 칩(Badge) 그리드를 탑재하여 대시보드 로드 및 수집 완료 시 자동 실시간 갱신 적용.
