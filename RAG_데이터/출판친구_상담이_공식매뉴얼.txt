# 출판친구 B2B ERP 통합 공식 상세 매뉴얼 v1.0

본 매뉴얼은 '출판친구' 플랫폼을 이용하는 출판사, 인쇄소 및 시스템 관리자들을 위한 공식 실무 운영 가이드입니다. 시스템에 구현된 실제 UI 기능, 인쇄 사양 기준, 단가 정책 및 챗봇 연동 규정을 상세하게 안내합니다.

---

## 1. 🏢 출판사(Publisher) 사용자 매뉴얼

출판사 회원은 도서의 인쇄 제작 사양 설정, 단가 시뮬레이션, 주문 발주, 배송지 분할 지정 및 정산 현황 조회를 직접 수행할 수 있습니다.

### 1.1. 도서 주문 및 사양 설정 프로세스
1. **인쇄 방식 선택 (생산 인프라)**
   * **[디지털 낱장 인쇄 주문]**: 소량 제작(50부 미만)에 적합한 인쇄 방식입니다.
   * **[디지털 연속지 인쇄 주문]**: 대량 제작(50부 이상)에 최적화된 회전식 고속 연속지 인쇄 방식입니다. 부수 구간별로 유리한 슬라이딩 단가가 자동으로 계산됩니다.
2. **상세 제작 사양 설정**
   * **규격(판형)**: 신국판(152×225mm) 및 A5(148×210mm), 국배판(210×297mm) 등을 지원합니다. (※ 신국판은 면당 12원, A5는 면당 8원의 기본 단가가 적용되며 판형 전환 시 최대 33.3%의 원가 절감 효과를 기대할 수 있습니다.)
   * **내지 인쇄**: 흑백 단면/양면, 부분컬러 단면/양면, 전체컬러 단면/양면 중에서 선택합니다.
   * **용지 종류**: 미색 모조(80g/100g), 백색 모조(80g/100g), 친환경 이코노미지 등을 선택할 수 있습니다.
   * **제본 방식**: 무선 제본(일반 단행본), 양장 제본(고급서적), 중철 제본(얇은 소책자)을 지원합니다.
   * **표지 코팅**: 무광 코팅(차분하고 고급스러운 재질), 유광 코팅(선명하고 화려한 광택) 중 선택합니다.
3. **인쇄용 PDF 파일 업로드 및 가이드**
   * **허용 포맷**: 내지와 표지 파일은 반드시 **PDF 파일 형식(`.pdf`)**이어야 합니다. 한글(`.hwp`), 워드(`.docx`), 이미지 파일 등은 자동 인쇄 조판 시스템과의 호환 문제로 접수가 불가합니다.
   * **폰트 아웃라인 처리**: PDF 출력 시 폰트가 깨지거나 대체되는 문제를 방지하기 위해, 모든 서체는 반드시 **'곡선 처리(Outlines)' 또는 '폰트 임베딩(Fonts Embedded)'**이 완료된 상태여야 합니다.
4. **다중 배송지 분할 입력**
   * 한 번의 주문으로 여러 배송처(서점, 물류창고, 저자 등)에 분할 배송할 수 있습니다.
   * [배송지 추가] 버튼을 클릭해 Daum 우편번호 서비스를 통해 정확한 주소를 검색하고, 각 배송처로 전달될 도서 수량을 나누어 지정합니다.

### 1.2. 주문 복제 및 정보 관리 ('불러오기' 기능)
1. **주문 불러오기 (복제 기능)**
   * 주문서 입력 창 상단의 **[불러오기]** 버튼을 누르면, 과거에 진행했거나 완료했던 주문 중 원하는 건의 인쇄 사양(지질, 규격, 색도, 제본 등)을 1초 만에 그대로 가져와 현재 주문서에 입력할 수 있습니다.
   * 매번 번거롭게 사양을 재입력할 필요 없이, 수량과 신규 배송지, 새로 수정된 PDF 파일만 지정하여 빠른 재발주가 가능합니다.
2. **정산 대시보드 조회**
   * [정산 및 주문관리] 탭에서 출판사 자사의 총 발주 부수, 누적 제작 지출액, 미수금 현황을 그래프 및 표로 모니터링합니다.
3. **발주서 엑셀 다운로드**
   * 주문 목록 옆의 **[발주서 다운로드]** 버튼을 클릭하면 ExcelJS 엔진을 통해 내부 결재 및 증빙용 공식 엑셀 파일이 즉시 렌더링되어 로컬로 다운로드됩니다.

## 2. 🖨️ 인쇄소(Printer) 사용자 매뉴얼

인쇄소 파트너는 배정된 신규 주문을 접수하고, 실제 인쇄/제본 공정 단계를 실시간으로 변경하며, 최종 운송장 번호를 입력하여 배송 처리를 완수합니다.

### 2.1. 생산 공정 제어 (칸반 보드 시스템)
* 인쇄소 대시보드 진입 시, 수신된 작업들이 단계별 카드로 나열됩니다. 작업자는 각 카드를 드래그 앤 드롭하거나 상태 선택 메뉴를 통해 실시간 공정 상태를 갱신합니다.
* **공정 단계**: `[접수 대기] ➔ [생산 진행] ➔ [인쇄/제본 중] ➔ [포장 대기/출고] ➔ [배송 완료]`
1. **작업지시서 및 PDF 확보**
   * 주문 카드를 클릭하여 [작업지시서 다운로드] 버튼으로 인쇄 규격 요약본을 수신합니다.
   * 상세 패널 내의 [내지 파일 다운로드] 및 [표지 파일 다운로드] 링크를 클릭해 출판사가 제출한 원본 PDF 파일을 내려받아 인쇄 장비에 로드합니다.
2. **출고 및 송장 등록**
   * 제본 및 포장이 완료되면 카드를 [포장 대기/출고] 단계로 이동시킵니다.
   * [송장 등록] 버튼을 눌러 이용한 택배사명과 정확한 운송장번호를 입력한 뒤 [출고 처리 완료]를 클릭합니다. 이 정보는 출판사 대시보드와 고객 알림으로 실시간 동기화됩니다.

### 2.2. 인쇄소 정산 및 주문관리 (Settlement & Orders)
인쇄소 파트너는 [정산 및 주문관리] 탭을 활용하여 월별 실적을 분석하고, 대금 청구를 위한 증빙 자료를 다운로드합니다.
* **[대시보드 보기] 버튼**: 매출 및 주문 추이 분석 차트(Chart.js 연동)를 열거나 닫아 화면 공간을 효율적으로 활용할 수 있습니다.
* **[거래명세서 전체 다운로드] (downloadTransactionStatementExcel) 버튼**: 클릭 시 그동안 납품 완료한 누적 거래 명세서 데이터를 포함한 엑셀(.xlsx) 정산 증빙 파일을 1초 만에 즉시 로컬 PC에 저장합니다.
* **[월별 마감 정산 확정] (executeMonthlyClosing) 버튼**: 해당 월의 총 인쇄 부수와 매출 단가를 대조하여 수동 취소/수정을 잠금 처리하고 정산액을 최종 확정합니다. (마감 후에는 출판사나 인쇄소 모두 주문 변경이 불가능합니다.)
* **[목표 설정 (설정 아이콘 ⚙️)] (setMonthlyGoal) 버튼**: 월별 목표 매출액을 설정하여 대시보드에서 실시간 매출 달성률(%)을 차트와 게이지로 모니터링할 수 있습니다.

### 2.3. 인쇄소 정보관리 및 계정 제어 (Printer Management)
관리자(Admin)는 B2B ERP의 [인쇄소관리] 탭을 통해 협력 인쇄소 계정을 중앙 통제하고 권한을 부여합니다.
* **[+ 신규등록] (clearPrinterMgmtFields) 버튼**: 신규 협력 인쇄소 가입 시 사용하며, 클릭하면 우측 폼이 초기화되어 새로운 인쇄소 파트너 계정을 등록할 수 있습니다.
* **[인쇄소명 검색] (printerSearch) 입력창**: 등록된 많은 인쇄소 중 특정 업체를 초성 또는 이름 검색으로 빠르게 필터링하여 찾아줍니다.
* **[접속 아이디] (pr_id) 입력란**: 인쇄소 파트너가 출판친구 ERP 포털에 로그인할 때 사용할 고유 계정 ID를 설정합니다.
* **[비밀번호 관리 / 비밀번호 초기화] (resetPrinterPassword) 버튼**: 보안 사고 발생이나 비밀번호 분실 시, 해당 인쇄소 계정의 비밀번호를 임시 또는 기본 패스워드로 안전하게 강제 초기화(Reset)합니다.
* **[인쇄소 마스터 설정]**: 업체의 계약 단가 등급(A/B/C 등), 담당자 정보, 세금계산서 발행 이메일 주소를 매핑하여 주문 발주와 정산 메일이 자동으로 발송되는 기반 정보를 관리합니다.

## 3. 🚨 자주 발생하는 질문 및 문제 조치 가이드 (FAQ)

### Q1. '부분 컬러 페이지' 칸에 숫자가 입력되지 않고 비활성화되어 있어요.
* **원인**: 시스템 기본값이 흑백 인쇄 모드 상태로 설정되어 있기 때문입니다.
* **조치**: 주문서 하단의 **[내지 설정] ➔ [내지 인쇄]** 항목에서 인쇄 옵션을 **`부분컬러 단면`** 또는 **`부분컬러 양면`**으로 먼저 클릭하여 선택해 주십시오. 옵션 선택 즉시 [부분컬러페이지] 입력 칸이 활성화되어 컬러 인쇄를 진행할 페이지 수를 기입할 수 있습니다.

### Q2. 도서 파일 업로드 시 업로드 실패 메시지가 나타납니다.
* **원인**: 파일 포맷이 `.pdf`가 아니거나, 파일 용량이 서버 전송 제한을 초과했거나, PDF 내부에 편집 보안 락(암호)이 걸려있기 때문입니다.
* **조치**: 파일 확장자가 소문자 `.pdf`인지 확인하시고, 웹 최적화용 PDF(출판 품질 유지)로 용량을 압축한 뒤 재시도해 주십시오. 계속 오류가 나면 암호화된 PDF인지 보안 설정을 확인하시기 바랍니다.

### Q3. 이미 등록한 주문의 사양을 변경하거나 주문을 취소하고 싶어요.
* **원인**: 인쇄소에서 작업 확인 후 생산에 착수(`생산 진행` 상태 이상)하게 되면 시스템 혼선을 방지하기 위해 출판사단에서의 직접적인 수정/삭제가 자동 제한됩니다.
* **조치**: 주문 카드 상태가 `접수 대기` 상태인 경우에만 우측의 [수정/삭제] 버튼이 작동합니다. 이미 공정이 시작되었다면 화면 우측 하단의 상담이 챗봇에 **"주문 취소 및 사양 변경 요청"**을 입력하여 **[문의 접수(티켓) 생성]**을 진행해 주십시오. 운영자(대표님)가 긴급 승인 후 조치해 드립니다.

## 4. 🤖 CS 상담이 이용 가이드 및 관리자 이관(티켓) 프로세스

1. **상담이 에이전트의 역할**
   * 출판친구 B2B ERP 내부 챗봇인 **[17번 고객응대 상담이]**는 기밀 데이터(마스터 단가, 타사 주문 정보 등)에 직접 접근하지 않는 철저한 보안 장벽(External CS Memory) 내에서 본 매뉴얼과 공식 규정에 기반한 표준 답변만을 정직하게 제공합니다.
2. **매뉴얼 외 질문의 거절**
   * 본 매뉴얼에서 확인되지 않는 시스템 정책이나 타사 정보 노출 질문에 대해서는 **"확인되지 않은 기능/정보입니다"**라고 답변하여 AI 환각 및 기밀 유출을 원천 방어합니다.
3. **관리자 이관 (긴급 티켓 발급 및 에스컬레이션)**
   * 기술적 오류, 정산 대금 불일치, 또는 생산 긴급 중단 요청 등 AI 상담이 스스로 처리할 수 없는 고난도 예외 케이스를 인지할 경우, 상담이는 즉시 사용자에게 **"대표님(운영자)에게 즉시 전달할 티켓으로 접수해 드릴까요?"**라고 질문합니다.
   * 사용자가 **[예]**를 누르면 대화 내용과 요청 사항을 종합한 `TICKET_PENDING` 로그가 Supabase DB에 적재되며, 16번 지휘_판다(통제실) 배너에 적색 CS 긴급 경보로 실시간 노출되어 관리자가 즉시 개입하여 해결을 도와드립니다.
