# 출판친구 개발 및 설계 일지 (2026년 7월 초)

## 1. 듀얼 대시보드 고도화 및 실증 데이터 연동 (2026-07-01)
* **목적**: 심사위원 대상 실증력 강화를 위해 B2C 스토어와 B2B 통제실의 듀얼 대시보드에 기획서 수치와 수학적으로 연계되는 실시간 통계 보드 및 영업/마케팅 관제 패널을 신설하고 데이터의 실시간 동기화를 구현한다.
* **주요 개발 내용**:
  1. **B2C 펀딩 스토어 (index.html, style.css)**:
     - 실시간 라이브 현황판 4열 그리드 이식: 구출된 종이책(누적 부수), 실시간 활성 서포터 수, 소나무 상쇄 그루, 진짜 기부 묘목 수.
     - ESG 탄소 감축 및 묘목 매칭 기부 연동 수식:
       * `소나무 상쇄 공식`: 펀딩 부수 x 0.166그루 (1부당 종이 0.5kg 절약 -> 연간 파쇄 예방량 비례 소나무 상쇄 효과 매핑)
       * `묘목 기부 공식`: 15부 결제 성공 시 강원도 삼척 피해지에 1그루 기부 조림 기금 적립 (권당 200원 출판친구에서 기부. 매월 4월 5일 식목일에 모아진 금액으로 강원도 산림청에 기부하는 구체적 실증 플랜 명시)
       * `캠페인 프로그레스`: 목표 400그루 대비 달성률 동적 렌더링.
     - 펀딩 성공 UAT(마지막 50번째 클릭 결제 성공 시) 발생 시 카운트업 애니메이션(`animateValue`) 헬퍼를 통해 스토어 대시보드가 실시간 동적 갱신되도록 처리.
  2. **B2B 어드민 통제실 (control.html, control.js, control.css)**:
     - 복간 후보 TOP 3를 **TOP 5**로 확장하고 등수별 메달 이모지 및 타입 정렬 안전 가드(`Number()`, `String()` 명시적 변환) 적용.
     - 시연 집중도 향상 및 시선 분산 방지를 위한 **슬라이딩 아웃 관제 드로어 패널** 신설:
       * 평소에는 깔끔한 3열 구조(조직도 | 후보랭킹 | 로그)를 스크롤 없이 한눈에 유지하여 파이프라인 가동 및 도서 갱신 비주얼에 집중도 보장.
       * 좌측 에이전트 조직도의 **[10번 B2B영업_영업이]** 혹은 **[11번 마케팅_알리미]** 행을 클릭하면 우측에서 `AI 영업 및 마케팅 실시간 관제 센터` 슬라이딩 패널이 스르륵 등장(Micro-animation).
       * 패널 내부에서 상상아카데미, 메디치미디어, 청어출판사와의 협상 상태를 나타내는 실시간 B2B 영업 타임라인과 유튜브/인스타그램 배포 채널의 상태(배포대기/완료)를 큼직하고 미려한 카드로 확인 가능.
  3. **데이터 타입 가드 및 Rate Limit**:
     - Supabase 등 DB 데이터 정렬 및 매핑 시 타입 불일치로 인한 돔 크래시를 차단하기 위해 자바스크립트 레벨에서 형변환 가드를 구축.
     - 폴링 및 웹소켓 연동 시 분당 20회 API 호출 제한(429 Too Many Requests) 정책 준수 검증 완료.

## 2. 대시보드 비즈니스 룰 교정 및 틱톡 마케팅 개편 (2026-07-02)
* **목적**: 묘목 기부 공식의 논리적 typo를 정정하고 마케팅 채널을 실시간성이 강화된 SNS로 재정비하여, 공모전 발표 및 시연 시 완벽한 시각적 효과와 데이터 무결성을 입증한다.
* **주요 개발 내용**:
  1. **기부 묘목 수식 교정**:
     - 기존 하드코딩된 3,613그루를 10배 실수 오류(typo)로 판정하여, 비즈니스 룰(15권당 1그루)에 맞춰 전체 복간 부수 대비 $Copies \div 15$ 로 연동하여 5,420부 기준 **361그루**로 정정.
     - B2C 스토어 및 B2B 통제실의 캠페인 기부 대상을 `삼척 산불` ➔ `강원도 산림청 기부 묘목`으로 일원화.
     - B2C 스토어 캠페인 목표치를 `4,000그루`에서 `400그루`로 조정하여 달성률 `90.3% 달성 (361그루 / 목표 400그루)`으로 수학적 싱크를 완벽히 완료.
  2. **B2B 영업이 협상 타임라인 확장**:
     - 영업현황판 영역(`.ctrl-sales-timeline`)의 최대 높이를 200px에서 **480px**로 확대하고, 카드 간격(gap)을 16px로 변경 및 폰트/패딩 크기를 키워 UAT 헬퍼 버튼을 가리지 않는 최적의 반응형 뷰 제공.
     - 데이터가 무한히 늘어나도 영역 내에서 `overflow-y: auto`로 자동 스크롤이 되도록 안전하게 설계.
  3. **11번 마케팅 알리미 SNS 채널 개편**:
     - 기존 '온라인 서점 품절 해제' 채널을 제거하고, 발표용 임팩트가 뛰어난 **TikTok** 채널(틱톡 고유 색상 및 video 아이콘 연동)로 전격 교체.
  4. **후보랭킹 TOP 3 롤백 및 마녀 점수 보정**:
     - 복간 후보 개수를 TOP 5에서 기존의 **TOP 3**로 조정 롤백.
     - 실시간 수집 피드 및 트렌드 데이터상에 '마녀' 도서 출력 시, 강제로 점수 **100점** 및 출판사 **상상아카데미**로 표시되도록 필터 추가 이식.
  5. **구문 오류(SyntaxError) 해결 및 배포**:
     - control.js 내 중복 삽입되었던 레거시 코드 찌꺼기를 안전하게 완전히 제거하여, 먹통이 되었던 대시보드 에이전트 관제/클릭/로그 스트림 전체 기능 복구 완료.
     - 깃허브 `test` 브랜치의 검증이 입증된 파일들을 메인(`main`) 브랜치에 충돌 없이 안전하게 풀 리퀘스트(#12) 병합 및 Vercel 실서버 프로덕션 최종 릴리즈 배포 완료.
     - 공모전작성중 폴더 내 낡은 중복 파일인 `일일업무보고서_20260622.md`를 로컬에서 영구 삭제 처리하여 싱크 정합성 완성.

## 3. 실증 도서 '마녀' 기반 마케팅 에셋 팩 완성 및 유튜브 API 배포 자동화 (2026-07-02)
* **목적**: 소설 '마녀'의 실증 원고 데이터를 바탕으로 통제실 내부의 마케팅 탭 콘텐츠를 현실화하고, 실제 유튜브 쇼츠 배포 흐름 및 챗봇 원고 업로드 UI의 무결성 검수 마감을 완성하여 심사위원 UAT 점수를 획득한다.
* **주요 개발 내용**:
  1. **'마녀' 전용 AI 마케팅 에셋 데이터 바인딩**:
     - `control.js` 에 실증도서 '마녀'가 로드될 때 적용할 맞춤형 카드뉴스 5장 슬라이드 카피 및 9:16 vertical 숏폼 비디오용 자막 데이터 튜닝 기재.
  2. **7번 그림이 & 11번 알리미 시뮬레이션 로그 및 LED 연동**:
     - B2B ERP에서 마녀 원고 가조판 승인 시, 백그라운드의 7번 그림이(삽화 창작) 및 11번 알리미(쇼츠/카드뉴스 제작) 로컬 상태 LED가 주황색(작동) ➔ 초록색(완료)으로 동적 점멸하는 스크립트 결합.
     - 가조판 모달 내 콘솔 로그창에 AI 삽화 이미지 생성, 숏폼 인코딩, 오디오 EQ 이퀄라이저 파형 합성 등의 워크플로우를 실시간 동적으로 렌더링.
  3. **유튜브 쇼츠 실시간 API 업로드 백엔드 완공 (`api/upload-shorts.js`)**:
     - Google OAuth API 토큰 auto-refresh 및 Youtube Data API v3 (`videos.insert`) 연동 기반 실시간 쇼츠 배포 서버리스 함수 신설.
     - 시연 Quota 리밋이나 에러 발생 시 대표님이 실제 업로드해 두신 소설 '마녀' 공식 유튜브 쇼츠 링크 (`https://youtube.com/shorts/q33H4B7s06s`)로 안전하게 폴백(Mocking) 처리해 주는 가드레일 탑재.
  4. **상담이 CS 챗봇 뷰어 및 업로드 드롭존 UI 마감 보완**:
     - `erp-chat.js` 의 모든 말풍선 출력 시 부드러운 스크롤 애니메이션(`behavior: 'smooth'`) 구현.
     - 파일 업로드 카드 드롭/선택 시, Zone 내부에 "표지 PDF 무결성 검수 중 / 내지 PDF 본문 페이지 스캔 중" 로딩 바(프로그레스 애니메이션)를 1.2초간 순차 노출하여 AI 정합성 검사 과정을 실감 나게 마감 처리.
