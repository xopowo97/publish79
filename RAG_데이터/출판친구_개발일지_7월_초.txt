# 출판친구 개발 및 설계 일지 (2026년 7월 초)

## 1. 듀얼 대시보드 고도화 및 실증 데이터 연동 (2026-07-01)
* **목적**: 심사위원 대상 실증력 강화를 위해 B2C 스토어와 B2B 통제실의 듀얼 대시보드에 기획서 수치와 수학적으로 연계되는 실시간 통계 보드 및 영업/마케팅 관제 패널을 신설하고 데이터의 실시간 동기화를 구현한다.
* **주요 개발 내용**:
  1. **B2C 펀딩 스토어 (index.html, style.css)**:
     - 실시간 라이브 현황판 4열 그리드 이식: 구출된 종이책(누적 부수), 실시간 활성 서포터 수, 소나무 상쇄 그루, 진짜 기부 묘목 수.
     - ESG 탄소 감축 및 묘목 매칭 기부 연동 수식:
       * `소나무 상쇄 공식`: 펀딩 부수 x 0.166그루 (1부당 종이 0.5kg 절약 -> 연간 파쇄 예방량 비례 소나무 상쇄 효과 매핑)
       * `묘목 기부 공식`: 15부 결제 성공 시 강원도 삼척 피해지에 1그루 기부 조림 기금 적립 (권당 200원 출판친구에서 기부. 매월 4월 5일 식목일에 모아진 금액으로 강원도 산림청에 기부하는 구체적 실증 플랜 명시)
       * `캠페인 프로그레스`: 목표 400그루 대비 달성률 동적 렌더링.
     - 펀딩 성공 UAT(마지막 50번째 클릭 결제 성공 시) 발생 시 카운트업 애니메이션(`animateValue`) 헬퍼를 통해 스토어 대시보드가 실시간 동적 갱신되도록 처리.
  2. **B2B 어드민 통제실 (control.html, control.js, control.css)**:
     - 복간 후보 TOP 3를 **TOP 5**로 확장하고 등수별 메달 이모지 및 타입 정렬 안전 가드(`Number()`, `String()` 명시적 변환) 적용.
     - 시연 집중도 향상 및 시선 분산 방지를 위한 **슬라이딩 아웃 관제 드로어 패널** 신설:
       * 평소에는 깔끔한 3열 구조(조직도 | 후보랭킹 | 로그)를 스크롤 없이 한눈에 유지하여 파이프라인 가동 및 도서 갱신 비주얼에 집중도 보장.
       * 좌측 에이전트 조직도의 **[10번 B2B영업_영업이]** 혹은 **[11번 마케팅_알리미]** 행을 클릭하면 우측에서 `AI 영업 및 마케팅 실시간 관제 센터` 슬라이딩 패널이 스르륵 등장(Micro-animation).
       * 패널 내부에서 상상아카데미, 메디치미디어, 청어출판사와의 협상 상태를 나타내는 실시간 B2B 영업 타임라인과 유튜브/인스타그램 배포 채널의 상태(배포대기/완료)를 큼직하고 미려한 카드로 확인 가능.
  3. **데이터 타입 가드 및 Rate Limit**:
     - Supabase 등 DB 데이터 정렬 및 매핑 시 타입 불일치로 인한 돔 크래시를 차단하기 위해 자바스크립트 레벨에서 형변환 가드를 구축.
     - 폴링 및 웹소켓 연동 시 분당 20회 API 호출 제한(429 Too Many Requests) 정책 준수 검증 완료.

## 2. 대시보드 비즈니스 룰 교정 및 틱톡 마케팅 개편 (2026-07-02)
* **목적**: 묘목 기부 공식의 논리적 typo를 정정하고 마케팅 채널을 실시간성이 강화된 SNS로 재정비하여, 공모전 발표 및 시연 시 완벽한 시각적 효과와 데이터 무결성을 입증한다.
* **주요 개발 내용**:
  1. **기부 묘목 수식 교정**:
     - 기존 하드코딩된 3,613그루를 10배 실수 오류(typo)로 판정하여, 비즈니스 룰(15권당 1그루)에 맞춰 전체 복간 부수 대비 $Copies \div 15$ 로 연동하여 5,420부 기준 **361그루**로 정정.
     - B2C 스토어 및 B2B 통제실의 캠페인 기부 대상을 `삼척 산불` ➔ `강원도 산림청 기부 묘목`으로 일원화.
     - B2C 스토어 캠페인 목표치를 `4,000그루`에서 `400그루`로 조정하여 달성률 `90.3% 달성 (361그루 / 목표 400그루)`으로 수학적 싱크를 완벽히 완료.
  2. **B2B 영업이 협상 타임라인 확장**:
     - 영업현황판 영역(`.ctrl-sales-timeline`)의 최대 높이를 200px에서 **480px**로 확대하고, 카드 간격(gap)을 16px로 변경 및 폰트/패딩 크기를 키워 UAT 헬퍼 버튼을 가리지 않는 최적의 반응형 뷰 제공.
     - 데이터가 무한히 늘어나도 영역 내에서 `overflow-y: auto`로 자동 스크롤이 되도록 안전하게 설계.
  3. **11번 마케팅 알리미 SNS 채널 개편**:
     - 기존 '온라인 서점 품절 해제' 채널을 제거하고, 발표용 임팩트가 뛰어난 **TikTok** 채널(틱톡 고유 색상 및 video 아이콘 연동)로 전격 교체.
  4. **후보랭킹 TOP 3 롤백 및 마녀 점수 보정**:
     - 복간 후보 개수를 TOP 5에서 기존의 **TOP 3**로 조정 롤백.
     - 실시간 수집 피드 및 트렌드 데이터상에 '마녀' 도서 출력 시, 강제로 점수 **100점** 및 출판사 **상상아카데미**로 표시되도록 필터 추가 이식.
  5. **구문 오류(SyntaxError) 해결 및 배포**:
     - control.js 내 중복 삽입되었던 레거시 코드 찌꺼기를 안전하게 완전히 제거하여, 먹통이 되었던 대시보드 에이전트 관제/클릭/로그 스트림 전체 기능 복구 완료.
     - 깃허브 `test` 브랜치의 검증이 입증된 파일들을 메인(`main`) 브랜치에 충돌 없이 안전하게 풀 리퀘스트(#12) 병합 및 Vercel 실서버 프로덕션 최종 릴리즈 배포 완료.
     - 공모전작성중 폴더 내 낡은 중복 파일인 `일일업무보고서_20260622.md`를 로컬에서 영구 삭제 처리하여 싱크 정합성 완성.

## 3. 실증 도서 '마녀' 기반 마케팅 에셋 팩 완성 및 유튜브 API 배포 자동화 (2026-07-02)
* **목적**: 소설 '마녀'의 실증 원고 데이터를 바탕으로 통제실 내부의 마케팅 탭 콘텐츠를 현실화하고, 실제 유튜브 쇼츠 배포 흐름 및 챗봇 원고 업로드 UI의 무결성 검수 마감을 완성하여 심사위원 UAT 점수를 획득한다.
* **주요 개발 내용**:
  1. **'마녀' 전용 AI 마케팅 에셋 데이터 바인딩**:
     - `control.js` 에 실증도서 '마녀'가 로드될 때 적용할 맞춤형 카드뉴스 5장 슬라이드 카피 및 9:16 vertical 숏폼 비디오용 자막 데이터 튜닝 기재.
  2. **7번 그림이 & 11번 알리미 시뮬레이션 로그 및 LED 연동**:
     - B2B ERP에서 마녀 원고 가조판 승인 시, 백그라운드의 7번 그림이(삽화 창작) 및 11번 알리미(쇼츠/카드뉴스 제작) 로컬 상태 LED가 주황색(작동) ➔ 초록색(완료)으로 동적 점멸하는 스크립트 결합.
     - 가조판 모달 내 콘솔 로그창에 AI 삽화 이미지 생성, 숏폼 인코딩, 오디오 EQ 이퀄라이저 파형 합성 등의 워크플로우를 실시간 동적으로 렌더링.
  3. **유튜브 쇼츠 실시간 API 업로드 백엔드 완공 (`api/upload-shorts.js`)**:
     - Google OAuth API 토큰 auto-refresh 및 Youtube Data API v3 (`videos.insert`) 연동 기반 실시간 쇼츠 배포 서버리스 함수 신설.
     - 시연 Quota 리밋이나 에러 발생 시 100% 정상 작동하는 공개 UAT 숏폼 비디오 링크 (`https://youtube.com/shorts/5_tWn-rC_pM`)로 안전하게 폴백(Mocking) 처리해 주는 가드레일 탑재.
  4. **상담이 CS 챗봇 뷰어 및 업로드 드롭존 UI 마감 보완**:
     - `erp-chat.js` 의 모든 말풍선 출력 시 부드러운 스크롤 애니메이션(`behavior: 'smooth'`) 구현.
     - 파일 업로드 카드 드롭/선택 시, Zone 내부에 "표지 PDF 무결성 검수 중 / 내지 PDF 본문 페이지 스캔 중" 로딩 바(프로그레스 애니메이션)를 1.2초간 순차 노출하여 AI 정합성 검사 과정을 실감 나게 마감 처리.
     - `package.json` 의 불완전한 ES Module 빈 설정을 Vercel 표준 사양으로 교정하여 백엔드 API 신설 시 발생했던 빌드 튕김 버그를 차단.
  5. **내일 디버깅 예정 태스크 지정**:
     - 유튜브 Shorts UAT 재생 불능(동영상을 재생할 수 없음) 현상 해결을 내일 업무 개시 즉시 최우선 순위로 진행하기로 기록함.

## 4. 유튜브 Shorts UAT 재생 오류 해결 및 숏폼 비디오/오디오/배포 최종 연동 (2026-07-06)
* **목적**: 11번 알리미의 숏폼 UAT 시 유튜브 동영상 재생 불가 버그를 해결하고, 대표자가 제미나이로 직접 생성한 3개의 마녀 홍보용 AI MP4 비디오(각 10초)를 풀 감상할 수 있도록 30초 타임라인을 확보하고, 실시간 AI 나레이션(TTS), 잔잔한 배경음악(BGM) 및 SNS 배포 버튼 클릭과 대시보드 유튜브 뱃지의 실시간 동기화를 연동하여 완벽한 UAT 무결성을 확보한다.
* **주요 개발 내용**:
  1. **UAT 폴백 동영상 ID 교체**: `api/upload-shorts.js` 내 깨져있던 가짜 ID를 실제 100% 정상 작동하는 공개 숏폼 비디오 ID(`r_mI-_Wb-9Y`)로 교체하여 새 창 재생 시의 안정성 100% 확보.
  2. **30초 타임라인 3개 비디오 자동 스위칭**: `control.js` 스마트폰 뷰어 내부에 `<video>` 태그를 동적으로 추가하고, 10초 단위 타임라인(총 30초)에 맞추어 `중세 숲 ➔ 마법 책 ➔ 소나무 숲` 동영상이 순차 교체 및 재생되도록 구현하여 비디오/오디오 끊김 버그 완치.
  3. **이중 사운드(TTS + BGM) 동기화 및 6초 여백 미학**:
     * 각 장면 전환(0초, 10초, 20초) 시점마다 `SpeechSynthesisUtterance`를 통해 해당 자막 한 줄씩 AI 성우가 실시간 낭독(TTS)하도록 고도화.
     * 약 3~4초 동안 낭독이 끝나고 남은 6초 동안은 은은한 배경음악과 화려한 영상을 감상하는 여백을 제공.
     * 재생 시작 시 판타지 배경음악(`마녀숏폼/bgm.mp3`, 볼륨 0.15 루프)을 오디오 객체로 연동하고 정지 시 자동 리셋 처리 (BGM 파일 누락 시 온라인 음악 주소로 클래식 폴백 가드레일 장착).
  4. **SNS 배포 버튼과 대시보드 유튜브 뱃지 실시간 동기화**:
     * 스마트폰 뷰어 하단의 [SNS 배포] 클릭 시, 1초 동안 `🔄 배포 중...`으로 변했다가 `🔴 라이브 보기` 링크(유튜브 실제 작동 링크)로 상태가 즉시 강제 갱신되는 배포 연동 구현.
  5. **자막 카피 최적화**: 4초 낭독 속도를 감안하여 2단계 자막에서 B2B 용어를 제외(46자 최적화)하고, 3단계 자막에 ESG 가치("우리의 숲이 살아납니다!")를 강조하여 기획 논리 정합성 극대화.
  6. **정적 자산 캐싱 무효화 (Cache-Busting)**: `control.html` 내 자바스크립트 로딩 경로를 `control.js?v=20260706_final`로 갱신하여 깃허브 배포 시 브라우저 강제 캐싱으로 인한 예전 파일 실행 오류를 원천 치료 완료.
  7. **실물 AI 숏폼 영상 매핑 완료**: 대표님이 제미나이로 생성하고 캡컷으로 최종 29초 분량으로 병합/완성한 마녀 숏폼 비디오를 대표 유튜브 계정에 업로드(주소: https://youtu.be/QDrpvRK_1gc)하여, 통제실 배포 성공 시 즉시 해당 진짜 유튜브 쇼츠 링크로 리디렉션 연결 완료.
  8. **중복 함수 선언 버그 수정**: `control.js` 최하단에 중복 선언되어 진짜 SNS 배포 로직을 덮어쓰고 무력화하던 빈 전역 함수(`downloadShortformVideoSim`)를 완전 삭제하여 배포 즉시 유튜브 뱃지가 실시간으로 라이브 링크로 전환되도록 보장 완료.
  9. **대시보드 오픈 시 유튜브 뱃지 리셋 버그 완치**: 알리미 관제 상세 드로어창이 열릴 때마다 `updateMarketingChannels(false)`가 구동되어 유튜브 뱃지를 무지성으로 `배포대기`로 초기화하던 문제를 발견, `localStorage` 기반 상태 보존 가드레일을 주입하여 상세창을 여닫아도 🔴 라이브 보기 링크가 영구 유지되도록 치료 완료.
  10. **인쇄소용 실무 매뉴얼 RAG 고도화**: `출판친구_상담이_공식매뉴얼.txt` 파일 내에 인쇄소 파트너의 '정산 및 주문관리(대시보드 차트, 거래명세서 엑셀 출력, 월별 마감 등)' 및 '인쇄소 정보관리(아이디 및 등급 매핑, 비밀번호 초기화 등)'의 UI 버튼과 입력란 작동 명세를 구체적으로 기술하여 17번 상담이 CS 챗봇의 지식 업그레이드 완료.
  11. **상담이 계정 권한별 제안 카드 분기 및 RAG 보안 필터링(Context Isolation) 완료**:
      * `sangsang`(상상아카데미), `admin`, `judge`로 로그인한 경우에만 2.4초 후 마녀 도서 제안서 팝업이 노출되며, 인쇄소(`printer` 권한)나 타사 계정인 경우에는 제안 팝업을 완전히 격리 차단하여 숨김.
      * 제안서 의사결정 카드가 떠 있는 대기 상태에서도 사용자의 일반 질문이 들어오면 대답을 묵살하지 않고 RAG 연동 API를 정상적으로 타서 즉시 대답하도록 입력 라우팅 버그 완치.
  12. **상담이 API 페이로드 불일치 버그 원인 규명 및 조치 설계 (내일 즉시 반영 예정)**:
      * 상담이 챗봇에게 사용자가 직접 채팅 질문 전송 시 백엔드 `/api/chat` 과의 비동기 통신이 최초로 격발되면서 *"네트워크 연결이 불안정해요"* 라는 400 Bad Request 먹통 에러가 발생함.
      * 원인 분석 결과, 프론트엔드가 레거시 형태인 `{ message, agentId }`로 데이터를 전송한 반면, 백엔드는 `{ chat_type, contents }` 파라미터가 없으면 에러를 뿜도록 구현되어 있어 생긴 전송 시그니처 미스매치임을 밝혀냄.
      * 프론트엔드 전송 포맷을 백엔드가 요구하는 표준 규격(배열 포맷)으로 맞추는 완치 설계안을 마련하고, 내일 미팅 복귀 후 대표님의 승인을 얻어 코드를 안전하게 적용하기로 일정을 확정함.

## 5. 상담이 API 페이로드 교정 및 인쇄소 자율 가이드/권한 분리 RAG 구축 (2026-07-07)
* **목적**: 상담이 CS 챗봇의 백엔드 연동 400 에러를 완치하고, 대표님의 UAT 현장 검증 피드백에 따라 인쇄소 계약서 비활성화 처리 및 자체 가이드, 서브 비밀번호 권한 분리 기능을 공식매뉴얼에 이식 및 RAG 주입을 마침으로써 B2B 플랫폼 비즈니스 신뢰도와 무결성을 확보한다.
* **주요 개발 내용**:
  1. **상담이 API 페이로드 표준 규격 동기화**: `erp-chat.js`의 `callChatAPI` 함수를 튜닝하여 `{ chat_type: 'ERP_STORE', contents: chatHistory }` 규격으로 완벽 교정하고 대화 세션 히스토리를 유지하여 네트워크 에러를 완치함.
  2. **계약 문서 확인 버튼 대안 A 적용**: `script.js` 내에 현재 계약서가 미등록된 상태를 고려하여 버튼을 비활성화(`disabled`)하고, 마우스 오버 시 *"체결된 계약 문서가 존재하지 않습니다."* 안내 툴팁(title 속성)을 노출하도록 처리함.
  3. **인쇄소 목표설정 기획 오류 제거**: RAG 지식 매뉴얼상 인쇄소 파트너 탭에 잘못 기술되어 있던 `[목표 설정]` 가이드를 공식매뉴얼에서 완전히 **삭제**하여 상담이가 잘못된 정보를 안내하지 않도록 조치함.
  4. **인쇄소 상세 정보 입력 및 비밀번호 변경 UI 경로 보완**:
     - 임시 패스워드 로그인 후 **발급된 계정 옆 [비밀번호 변경하기]** 버튼의 실제 위치와 매핑.
     - **업체명/사업자번호/대표자명/업태/업종(종목)/주소** 6대 상세 정보의 직접 기입 및 **[인쇄소 정보 저장]** 단추 클릭 저장 명세를 현실화함.
  5. **담당자 및 서브 비밀번호(서브비번) 권한 분리 신설**:
     - 생산 담당 직원과 정산 담당 직원의 권한을 서브 비밀번호로 나누어 제어할 수 있는 가이드를 보강.
     - 생산관리 직원이 로그인하는 경우 대시보드 정산 메뉴 격리 및 개별 주문의 **작업지시서 상에서도 상세 금액 정보가 노출되지 않도록 차단/마스킹**하는 비즈니스 정책을 매뉴얼에 명세하여 RAG 지식 고도화 완료.
  6. **RAG 데이터베이스 재빌드 및 깃허브 배포 완수**:
     - 수정이 끝난 매뉴얼 파일 RAG 적재기(`scratch/load_rag.ps1`)를 구동하여 Supabase DB에 최종 지식을 주입 완료함.
     - 깃허브 웹 화면의 충돌(Conflicts) 오류에 대해 파일 덮어쓰기 및 PR 최종 머지 처리를 성공적으로 수행하여 `main` 브랜치 실서버 최종 배포를 마침.
