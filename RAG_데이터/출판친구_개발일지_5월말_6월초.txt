# 10번_자가치유코딩에이전트.md

## 1. 에이전트 개요 (역할 및 목적)

* **직책**: 수석코드개발팀장 (Chief Code Development Officer)
* **목적**: 9번 기술행정지원 실장으로부터 전달받은 에러 패킷(Error Packet)을 수신하여, 플랫폼의 핵심 소스코드인 `script.js` 및 `api/` 내부의 결함을 식별하고, AI 모델(Gemini API)을 통해 실시간 Hot-Fix 패치 코드를 자율 생성 및 GitHub PR을 발행하여 시스템 영속성을 보장함.

## 2. 연동 상태 및 환경 변수 정보

* **상태**: 구현 완료 (Production Ready)
* **통신 API**: GitHub REST API (PR 생성, 브랜치 분기), Gemini API (코드 정밀 분석 및 패치 생성)
* **환경 변수**: `GITHUB_TOKEN`, `GEMINI_API_KEY` (Vercel Production 환경 변수 격리 관리)
* **데이터베이스 연동**: Supabase `deploy_status` 테이블(패치 상태 기록), `agent_audit_logs`(자가치유 활동 감사 로그)

## 3. 세부 구현 로드맵 (Step 1 ~ Step 4)

* **Step 1 (분석 단계)**: 9번 에이전트가 격리한 에러 발생 지점의 소스코드(전후 15라인)를 AST(추상 구문 트리) 기반으로 격리 분석.
* **Step 2 (패치 생성)**: 결함 원인(오타, 변수명 불일치 등)을 Gemini 모델에 질의하여, 기존 로직과의 정합성을 100% 보장하는 Hot-Fix 패치 문자열 자동 생성.
* **Step 3 (보안 검증 연동)**: 생성된 패치 코드를 즉시 12번 인프라 보안관에게 전달하여 민감 데이터 유출 여부를 30초 내 검증.
* **Step 4 (자율 반영)**: 보안 검증 완료(SAFE) 시, GitHub API를 통해 `patch/self-heal-*` 브랜치를 자동 생성하고 PR(Pull Request)을 발행하여 관리자(대표님)에게 보고.
